Project Looking Glassへの貢献

Project Looking Glassのソースコードは、ライセンス契約条件に従って、誰でも利用することができます。 このプロジェクトの開発は公の場でおこなわれていて、個人や企業に対してオープンです。 修正や拡張機能が実装されるよいプロジェクトによって、すべての人が利益を得ることができます。 Project　Looking GlassのようなJavaDesktopコミュニティプロジェクトに参加する一般的なガイドラインはJavaDesktop Community Governance Guidelinesを参照してください。

共同著作権契約書

ソースコード、オブジェクトコード、ドキュメント、または他の素材のすべての貢献者は、貢献がこのプロジェクトに採用される前に、共同著作権契約書フォームのすべての項目に記入しなければいけません。 印刷し、すべての項目に記入し、署名してから、共同著作権契約書フォームを下記にメールまたはFAXしてください。

貢献方法

以下は、Project Looking Glassに貢献する方法のいくつかです。

  • バイナリリリースの試用
    あなたは、Project Looking Glassのバイナリリリースをlg3d-coreプロジェクトのドキュメントセクションからダウンロードすることができます。

  • フォーラムおよびメーリングリストへの参加
    Project Looking Glassフォーラムとメーリングリストは、このソフトウェアを使う人たちにお互いの専門知識の恩恵を受けるための手段を提供します。 本当に便利にするために、これらのリストはあなたの入力を必要としています! どうか怖がらないで、投稿してください。 たとえば、可能ならば他の人を助けたり、質問、コメントおよびあなたが良いと思うアイデアを投稿してみてください。 詳しくは、lg3d-coreページの乗り遅れないためにのセクションを参照してください。

  • バグレポートの登録
    開発者にとって、バグレポートは非常に役に立つので、プロジェクトに貢献するための素晴らしい方法です! バグ(欠陥)を登録するために、lg3d-coreプロジェクトの課題追跡を使ってください。 バグレポートを記入する時には、少し時間が掛かることを覚えておいてください。 その問題を再現できることを確認して、その問題についての説明を非常に明白に述べてください。

  • 拡張要求の登録
    もしプロジェクトにあってほしいものやいらないもの、または改良できると思われるものがあるのならば、拡張要求を記入して私たちに知らせてください。 課題追跡を使って、"Defect"ではなく"Enhancement"で登録してください。

  • lg3d-coreへのコードの貢献
    すでに述べたように、ソースコードのすべてが利用できます。 もし欲しい機能や修正して欲しいバグがあるのならば、あなたはそれを実現することができます。 直接ソースコードに貢献するためには、このプロジェクトのDeveloperロールを持っている必要があります。 しかし、それをおこなう前に、まずプロジェクトのObserverになって、ソースコードをダウンロードして、APIの内部に習熟してください。 一番最初に貢献する前に、共同著作権契約書を提出することを忘れてはいけません。

    私たちがおこなっている典型的なコード更新手続きは、次の通りです。

    1. 課題追跡を使って、バグや拡張要求を登録してください。
    2. その課題のページの"追加コメント"セクションを使って、修正について議論してください。 課題ページにパッチを貼り付けるのは、いい考えです。 もしその課題について、より多くの人たちと議論したいのなら、 Project Looking Glassフォーラムを使ってください。
    3. いったん私たち(少なくとも、プロジェクトのオーナーの一人)がその変更に同意すると、現在の開発用ブランチ(注: それはHEADではありません)にそれをコミットしてください。

    私たちも時々これらのステップを忠実におこなわないのですが、コードベースとその変更に慣れるまでは、これらに従うことを勧めます。 どのように物事が進んでいるのかを見るために、 このページからcvs@lg3d-core.dev.java.netとissues@lg3d-core.dev.java.netエイリアスに登録することもできます。

  • IncubatorにおけるLG3Dアプリケーションの貢献
    Lg3d-incubatorは、あなたのクールなLG3Dアプリケーションを共有するための場所です! あなたが望むのならば、それをコミュニティと一緒に育てることができます。 あなたが新しいLG3Dアプリケーション開発を開始したいのならば、励みになるようにあなたのアイデアを Project Looking Glassフォーラムに投稿してみてください。 あなたは、多くの有益なフィードバックが得られるでしょう。 あなたのコードをアップロードして、コミュニティと共有するためには、共同著作権契約書をまだ提出していなければ、共同著作権契約書を提出する必要があります。 その後、IncubatorプロジェクトにおけるDeveloperロールが認められるでしょう。

    注: あなたのコードを登録する時には、あなたのアプリケーションが次のリリースに問題なく取り込めるように、(HEADブランチに対してではなく)現在の開発ブランチに対しておこなってください。

  • Webコンテンツの貢献
    私たちは、Webコンテンツを充実させるための助けを探しています。 コンテンツに直接貢献するためには、このプロジェクトのContent Developer (または Developer)ロールを持っている必要があります。 ロールを申し込む前に、共同著作権契約書を提出するのを忘れないでください。

  • 翻訳
    あなたは、ドキュメント作成と同様に、これらのページの翻訳を支援することで手伝うこともできます。 これらのページをlg3d-coreフロントページ上で見ることができるいくつかの言語への翻訳作業は、すでにおこなわれています。 私たちは、世界的なLG3Dコミュニティを育てることができるように翻訳を他の言語に対しても広げたいと思っています。 これをおこなうためには、Content Developer (またはDeveloper)ロールが必要です。 この手続きは、他のコードの貢献のための署名と同じです。

    • もしあなたが 進行中として列挙されているサブプロジェクト 以外の言語に対して新しい翻訳プロジェクトを開始したいのならば、 まず最初に Project Looking Glassフォーラムで関心があることを表明してください。
    • もしあなたが進行中の作業に加わることに興味があるのならば、 ここのリスト をチェックして、そこのリーダにコンタクトするか、フォーラムで関心があることを表明してください。

    ファイルフォーマットや翻訳作業を楽にするための他の良いアイデアなどの詳しいガイドラインについては翻訳ファイルスタイルガイドを参照してください。

    注: Webコンテンツを反映する時には、あなたのおこなった更新がすぐ反映されるように、(現在開発中のブランチではなく)HEADに対しておこなってください。
