LG3D の概要

川原 英哉 KAWAHARA Hideya

はじめに

Project Looking Glass とは?

Project Looking Glass注1 は，次世代デスクトップおよびユーザインターフェースを探求するオープンソースプロジェクトです．

その目的は，3 次元 (3D) 描画やアニメーションを活用し，表現力と使い勝手を向上することです．もともと，LG3D は Sun Microsystems, Inc.(以下
Sun) の先進開発グループが推進していたプロジェクトでしたが，「技術革新をオープンソースコミュニティの手で」を合言葉に，2004 年 6 月にオープン
ソース化したものです．

本特集では，読者の皆さんを LG3D の最新技術の世界へご案内いたします．

 

注 1 Project Looking Glass は LG3D とも呼ばれています．本稿では，原則的に LG3D と表記します．

 

何がそんなにおもしろいの?

図 1 LG3D プロジェクトの日本語 Web サイト (https://lg3d-core.dev.java.net/ja/)
3D の利点を効果的に利用

今日，PC の性能は飛躍的に向上しました．とくに 3D 描画の性能の向上には目を見張ります．ところが，PC のデスクトップやユーザインターフェー
ス (UI) は，Apple の初代 Macintosh が発売されて以来 20 余年，本質的には変化していないと言えるのではないでしょうか．「何かできるはずだ」「何かしてみたい」とは思いませんか．LG3D も同じ想いを元にスタートしました．

3D を活用したデスクトップやUI に関する研究は，新しい話ではありません．ただ，操作性を無視して，バーチャルリアリティの世界にデスクトップを無意味に引きずり込もうとしていたものが多く見受けられました．研究色の強いプロジェクトも多かったと思います．

LG3D は，あえて既存のデスクトップの UI を部分的に引き継ぎつつ，効果的に 3D が利用できると思われる点にだけ注力して，その部分に積極的に 3D を適用するというアプローチを取りました．この点が，LG3D がこれほど多くの人の支持を受けた大きな理由の 1 つであるという意見をよく耳にします．

オープンソース & 活発な活動

また，LG3D はオープンソースであり，誰でも開発に参加できます．うれしいことに，プロジェクトの Web サイトには日本語版もあります (図 1)．

過去 20 年間，本質的に停滞していたデスクトップや UI の革新に誰でも参加できる，この点が，LG3D の魅力の1つではないでしょうか．

図 2 Windows 上で Java Web Start 版 LG3D を実行したところ
Java Web Start 版のLG3D

とりあえず少しでも自分で動かしたい，という方は，Java Web Start 版の LG3D を試されても良いかもしれません．すべての機能が使えるわけではないのですが，複雑なインストールなしにその雰囲気を手軽に味わうことができます注2．

利用するためには，まず使用する PC が 3D アクセラレータを搭載しており，3D 描画性能が出るようにきちんと設定されていることを確認してください． Windows マシンで試すのが一番お手軽でしょう．

まず，JRE 5.0 もしくは JDK 5.0 がインストールされているか確認し，なければインストールしてください．続いて，https://lg3d-webstart.dev.java.net/ に Web ブラウザでアクセスし，「Downloads」の項目の「Core demo」もしくは「Core and Incubator demo」をクリックするだけで LG3D のウィンドウが開きます注3．

図 2 は Windows 上で Java Web Start 版 LG3D を実行したものです．まずは，手当たり次第にいじってみてください注4．

 

注 2 「認証」の扱いが甘いのが玉に瑕ですが．

注 3 1 度実行した後は Java Web Start にキャッシュされますので，次回からはダウンロードを待つことなく起動できます．

注 4 使い方の詳しい説明は本章の「LG3D とアプリの使い方/同梱アプリの説明」をご覧ください．なお，描画が極端に遅い場合は，3D アクセラレータの設定不備か 3D アクセラレータの性能不足の可能性があります．

 

きちんと動かすには何が必要?

既存アプリを 3D 空間内で動かすなど，LG3D の機能のすべてを利用するためには，Linux もしくは Solaris x86 が必要です．しかし，機能は限定されてしまうのですが，Windows でも動作させることは可能です．インストールの詳細と LG3D の使用方法は，この後で詳しく説明します．

 

どうやってプログラミングするの？

LG3D は，「次世代デスクトップおよびユーザインターフェースを探求するオープンソースプロジェクト」です．オープンソースというからには，皆さんにもどしどし開発に参加していただきたいのです．もちろんそのために，3D アプリケーションを開発するための API が用意してあります．これについては，2 章で詳しく説明します．

 

コミュニティ活動に参加するには?

新しい 3D UI のアイデアが沸いてきたり，とても Cool な 3D アプリケーションを開発したら，それを多くの人に見せて意見を聞いてみてはいかがでし
ょうか．LG3D には活発なコミュニティが存在します．その活動への参加の第一歩について，4 章で説明します．コミュニティ活動に参加されている
方々の生の声もお届けします．

 

LG3D のインストール

藤槻泰宏　FUJITSUKI Yasuhiro http://blogs.sun.com/thaniwa

はじめに

非常に Cool な次世代デスクトップ環境「LG3D」ですが，写真などで見るだけではあまりおもしろくありません．実際に動かしてみましょう! そして，LG3D の世界へどっぷりはまりましょう!

ということで，LG3D 導入の第 1 歩目となるインストール方法および起動方法を解説したいと思います．

 

インストールの前に

LG3D の動作プラットフォーム

LG3D のサポートプラットフォームは Linux および Solaris 10 x86 です注5．また，Windows でも一部のモードを利用することができます．

LG3D を動かすためには 24 ビットカラーおよび OpenGL 1.3 をサポートしたグラフィックス環境が必要になります．

Linux では，

  • NVIDIA 製 GeForce シリーズ
  • ATI 製の Radeon 8500 以降

Solaris 10 x86 では，

  • NVIDIA 製 GeForce シリーズ，Quadro シリーズ

のようなグラフィックスカードを搭載した PC が必要になります．

 

注 5 本稿の内容は Fedora Core 3 および Sun Java Desktop System でテストしましたが，他のディストリビューションでも同様の方法でインストール可能です．

 

LG3D で必要となるコンポーネント

LG3D を実行するためには，あらかじめ次のコンポーネントをインストールしておく必要があります注6．

  • JDK 5.0 もしくは JRE 5.0 以降 (本稿執筆時点の最新版: JDK 5.0 Update 4)
  • Java3D 1.4.0注7 (本稿執筆時点の最新版: Java3D 1.4.0 Beta5)
  • Java Advanced Imaging AP （JAI) 1.1.2_01 以降

以降で，LG3D のインストールと併せて解説します．

 

注 6 本稿では，Release 0.7.0 を対象にしています．

注 7 Java3D 1.4.0 は現在ベータ版ですので，できるだけ最新のものをご利用ください．

 

LG3D のインストール

Linux，Solaris 10 x86 における LG3D のインストール手順は次のとおりです．

  1. グラフィックスドライバのインストールおよび 3D アクセラレーションの設定
  2. Java 実行環境 (JDK もしくは JRE)注8 および Java3D，JAI のインストール
  3. LG3D バイナリのインストール

 

注 8 本稿では JDK を用いたインストール方法についての解説を行います．

 

グラフィックスドライバのインストール

3D アクセラレーション機能を利用するために，各メーカが提供するドライバをインストールする必要があります．これらのドライバはディストリビューションには含まれていないため，各メーカの Web サイトからダウンロードします．

Linux 環境では，ドライバのインストールには GCC などの開発環境および kernel-source (もしくは kernel-headers) が必要となりますので，あらか
じめインストールしておきます注9．

ここからは，ドライバのインストール方法を環境ごとにまとめます．

 

注 9 NVIDIA 製ドライバのインストールの場合，Fedora Core 3 など一部のディストリビューションでは kernel-source が不要です．

 

ATI 製ドライバのインストール (Linux 編)

ATI 製のグラフィックスカードを利用している場合は，ATI の Web サイトからドライバをダウンロードします．http://www.ati.com/jp/ から，「ドライバー」→「Linux Drivers and Software」→「RADEON 8500 Series and higher」とリンクをたどっていくと見つかります．

 

XFree86/Xorg のバージョンを確認

ATI 製のドライバは，XFree86/Xorg のバージョンごとに用意されていますので，ダウンロードを行う前に XFree86/Xorg のバージョンを確認しておきます．

XFree86/Xorg のバージョンは端末エミュレータで，図 3 (1) のようにすることで確認できます．自分の環境に合ったものを選択してください．本稿では Xorg 6.8 用のドライバ (fglrx_6_8_0-8.16.20-1.i386.rpm) を利用します．

 

ドライバのインストール

ドライバのインストール作業は root ユーザで行います．ドライバのインストールを行う前に X を終了し，コンソールモードに移動しておきます (図 3 (2))．

コンソールモードに移動したら，root ユーザで再度ログインし，rpm コマンドを利用してドライバのインストールを行います注10 (図 3 (3))．ドライバをインストールした直後は /etc/X11/xorg.conf がないため X の起動が行えませんので， fglrxconfig を利用して /etc/X11/xorg.conf を作成します (図 3 (4))．

コンフィギュレーションを終了後，X の起動を行います (図 3 (5))．ドライバの設定が正しく行えた場合は，端末エミュレータで fglrxinfo を実行すること
により，OpenGL のバージョンなどを確認できます (図 3 (6))．

NVIDIA 製ドライバのインストール (Solaris 編)

Solaris 10 x86 用の NVIDIA 製グラフィックスドライバは，Linux と同様に NVIDIA の Web サイトで公開されています．

GeForce シリーズを利用する場合，グラフィックスカード情報をシステムに別途登録する必要があります (Quadro シリーズの場合は不要です）．GeForce FX5200 の場合，図 5 (1) のようになります．

NVIDIA 製ドライバのインストールは root ユーザで行います．端末エミュレータで NVIDIA 製のドライバファイルを実行することにより，ドライバのインストールが行えます (図 5 (2))．この時点では /etc/X11/xorg.conf がないため，/etc/X11/xorg.conf.nvidia というあらかじめ設定が行われているサンプルファイルを /etc/X11/xorg.conf にコピーします (図 5 (3))．

また，lg3d-session (後述) を利用したい場合は，Linux 用ドライバのインストールと同様に "AllowGLXWithComposite" オプションを有効にします．

ドライバのインストール終了後にリブートします (図 5 (4))．

Java 環境のセットアップ

インストール前の注意

Java 環境のセットアップの解説は Linux をベースに行います．バイナリパッケージは原稿執筆時点で最新のものを利用しています．

本稿では例として，インストールに必要なファイルのダウンロード先を /tmp，インストール先ディレクトリを /opt/lg3d とします．Java3D のベータ
版を利用しているので，Java 環境は LG3D 専用として /opt/lg3d/jdk1.5.0_04 以下にインストールするようにします．他のディレクトリにインストールし
たい場合は，ディレクトリを読み替えてください．

Solaris 10 x86 環境にインストールする場合は，ファイルを Solaris 10 x86 対応のもの注11に置き換えることにより，同様の方法でインストールを行うことが可能です．

 

注 11 jdk-1_5_0_04-solaris-i586.sh，java3d-1_4_0-build5-solaris-x86.zip，jai-1_1_2_01-lib-solaris-i586-jdk.bin の 3 ファイルです．

 

JDK のインストール

http://java.sun.com/j2se/1.5.0/download.jsp から JDK の Linux self-extracting file (jdk-1_5_0_04-linux-i586.bin) をダウンロードします．

ダウンロード後，インストール先のディレクトリ (/opt/lg3d) に移動し，ダウンロードしたファイルを実行します．すると，ライセンス表示が行われますので，ライセンスを受諾 (yes と入力) すれば JDK がインストールされます (図 6 (1))．

Java3D のインストール

https://java3d.dev.java.net/binary-builds.html からバイナリ (java3d-1_4_0-build5-linux-i586.zip) をダウンロードします．ダウンロードしたファイルを解凍すると，j3d-140-build5-linux-x86.zip ができます．このファイルを /opt/lg3d/jdk1.5.0_04/jre で解凍します (図 6 (2))．

JAI のインストール

http://java.sun.com/products/javamedia/jai/downloads/download-1_1_2_01.html から，JDK インストール用のバイナリ (jai-1_1_2_01-lib-linux-i586-jdk.bin) をダウンロードします．このファイルを JDK ディレクトリ (/opt/lg3d/jdk1.5.0_04) 内で実行します．ライセンス表示が行われますので
yes と入力します (図 6 (3))．

LG3D 基本動作

藤槻さんの「LG3D のインストール」を参考にインストールが無事完了したら，さっそく LG3D を動かしてみましょう．ここでは便宜上，LG3D の API を使って開発された LG3D 専用のアプリケーションを「LG3D アプリ」，普通の X 環境用のネイティブアプリケーションを「X アプリ」，LG3D アプリと X アプリをまとめて「アプリ」と呼ぶことにします．LG3D アプリのプログラミングについては 2，3 章で詳しく説明されていますので，楽しみにしていてください．

 

LG3D の基本コンポーネント

図 11 LG3D 起動直後の画面

LG3Dを起動すると，図11のような画面が現れます．

タスクバーとアイコン

画面の下のほうに，緑色の半透明なタスクバーがあります．その上のアイコンにマウスポインタを重ねると，アイコンは少し大きくなって浮き上がり，選択しやすくなります．

アイコンは 4 種類に分けることができます (図 12)．

右端のドクロ印の海賊旗アイコンを左クリックすると LG3D が終了します．左端の PC のアイコンは xterm (端末エミュレータ) 起動用のものです．xterm は X アプリなので，実行環境に制限があります．詳細は，「X アプリの実行」を参照してください．

その他のアイコンを左クリックすると，背景設定メニューまたは LG3D パッケージに含まれている LG3D アプリが起動します．背景設定メニューと，同梱の LG3D アプリの具体的な操作方法については，それぞれ「背景の設定」，「同梱アプリの紹介」で説明します．

また，タスクバーはただのアイコン置き場ではなく，背景操作のための機能を持っています．それについては，「背景の設定」中の「パノラマ背景」で紹介します．

背景の設定

図 15 背景選択メニュー
図 16 仮想デスクトップ全体の縮小表示画面

タスクバーの右端から 2 つ目のアイコンを左クリックすると，背景選択メニューが表示されます (図 15)．「OK」ボタンをクリックすると，一番手前に表示されている画像が背景として設定されます．

背景の種類

背景には，

  • Panoramic (パノラマ背景)
  • Image (イメージ背景)
  • Layered (レイヤード背景)

の 3 種類があります．デフォルトではメニューの一番上にある「All」が選択されており，パッケージに含まれているすべての背景が選択用のリング状
の一覧から参照可能です．

パノラマ背景

「Panoramic」を選択すると，仮想デスクトップモードで使用可能な背景だけが選択肢として表示されます．このモードは，横長の背景を持つ仮想デ
スクトップの一部だけを画面に表示してデスクトップとして使用するものです．

パノラマ背景使用時には，タスクバー上で右クリックすると仮想デスクトップ全体が縮小表示されます (図 16)．そこで使用したいデスクトップを左クリックで選択することができます．

パノラマ背景使用時にデスクトップを切り替える方法はもう 1 つあります．画面の左右の端 (アプリのウィンドウが表示されていない部分) にマウスポインタを持っていったときにマウスポインタが左右の矢印の形になる場合には，そこには選択可能なデスクトップが存在しています．その状態でマウスを左クリックすると，そちら側のデスクトップに切り替わります．

イメージ背景

「Image」を選択すると，ごく普通の画像イメージの背景だけが選択肢として表示されます．

レイヤード背景

「Layered」を選択すると，奥行きのある階層的な背景だけが選択肢として表示されます．この背景使用時には，マウス操作に合わせて背景がゆらぎ，不思議な感覚が味わえるのですが，それは言葉で説明するよりも実際に体験していただいたほうが良さそうです．

 

ウィンドウの操作

図 17 奥行きのあるウィンドウ配置
図 18 画面端に片付けられたウィンドウ
図 19 踊る Zoetrope

いよいよ LG3D のウリの 1 つでもある，楽しいウィンドウ操作の説明に入ります．

画面上での移動

アプリのウィンドウ上でマウスポインタが上下左右の矢印の形になっているとき，そのウィンドウを動かすことができます．左ボタンでドラッグすることで，画面上を上下左右に，またホイールで手前から奥へ，奥から手前へウィンドウを移動させることができます．図 17 では，後ほど「同梱アプリの紹介」のところで紹介する LG3D アプリの 1 つである Zoetrope という名前の画像ビューアが中央の奥に小さく表示されており，右側には LG3D ヘルプのウィンドウの一部が手前に大きく表示されています．

邪魔なウィンドウを片付ける

今必要としていないアプリのウィンドウを，LG3D では「画面の端に立たせておく」ことで片付けることができます．

特定のウィンドウを片付けたいときには，そのウィンドウを画面の端に接するように移動させるか，中ボタンで片付けたい方向にドラッグします．

また，すべてのウィンドウを片付けるには，ウィンドウが表示されていない部分で右クリックします (図 18)．

片付けたウィンドウの復活/反対側の隅への移動

いったん片付けた (立たせた) ウィンドウの中から，必要なものだけを復活させる方法は 2 つあります．

  • 立っているウィンドウを左クリック
  • サムネイルを左クリック

また，一括して片付けたウィンドウすべてを一度に復活させるには，片付けられたウィンドウがある側 (画面の左右どちらか半分) の背景で右ク
リックします．逆に，反対側の背景で右クリックすると，すべてのウィンドウがそちら側に片付けられます．

Java ロゴでウィンドウ回転

通常，背景上のロゴというのは飾りに過ぎないものですが，遊び心あふれる LG3D では違います．画面右上の Java ロゴをマウスの左ボタンでドラッ
グすると，それに合わせて画面上に表示されているアプリのウィンドウが 3D 空間内で回転します．図 19 は，Zoetrope 画像ビューアを回転させている
ところで，図ではわかりにくいかもしれませんが，表示されている画像が上下逆の裏返しになっています注13．

 

注 13 なお，この機能は「パノラマ背景使用時で，仮想デスクトップ全体の縮小表示を行っているとき」「イメージ背景使用時」
には使えません．

 

X アプリの実行

藤槻さんの「LG3D のインストール」「LG3D の実行」で説明されているとおり，X アプリは lg3d-app または lg3d-session コマンドで立ち上げた LG3D 上でのみ使用可能です．

「LG3D の基本コンポーネント」で述べたように，タスクバーの左端にあるアイコンを左クリックすると，X アプリの 1 つである「xterm (端末エミュレータ)」が立ち上がります．この xterm から，他の X アプリを起動してみましょう．

筆者が LG3D を動かしている環境は SUSE Linux 9.1と Solaris 10 です．その上で実行した LG3D では,

  • Firefox 1.0.6
  • OpenOffice.org 1.1.4

などのアプリを動かすことができました．しかし，LG3D はまだ開発途上であり，X アプリの中には起動できないものや異常終了するものがあるのは事
実です．このあたりの X 絡みの問題は今後徐々に解決されていくはずです注14．

 

注 14 LG3D の改良に興味をお持ちの方，力を貸してくださる方は，ぜひコミュニティに参加お願いしますっ！

LG3D の基本操作と同梱アプリの紹介をさせていただきました．ここに登場したもののほとんどは，コミュニティのメンバの手によるものです．こういった活動に興味をお持ちの方は，ぜひメンバに加わって，この楽しさを体験していただければと思います．

Stable builds と Daily builds

ここまでは，Stable builds である LG3D 0.7.0 Release (6 月 24 日にリリースされた Release 0-7-0) のパッケージを使って説明をしてきました．ところで，同じ場所注から開発中の次バージョン 0.7.1 の Daily builds をダウンロードできます．図 A は 8 月 9 日版の 0.7.1 のビルドでの LG3D 起動画面のタスクバーの部分で，0.7.0 と比べるとタスクバー上のアプリが 2 つほど増えていることがわかります．

このように LG3D はどんどん進化を続けています．現時点での正式安定リリース版 0.7.0 に慣れた後は，開発中の最新版 Daily builds で楽しんでいただくのも良いのではないかと思います．
