コミュニティの広場

コミュニティメンバが語る LG3D への想い

 

初出: Software Desing 2005 年 10 月号

ここでは，コミュニティ活動の一部を紹介するとともに，LG3D コミュニティメンバの方々の生の声を，読者の皆さんにお届けします．

活動の概要

川原 英哉 KAWAHARA Hideya

活発な LG3D コミュニティ

図 1 javadesktop.org フォーラムのページ．投稿数からも LG3D のフォーラムが一番活発なのがわかる

LG3D は，昨年の「JavaOne 2004 San Francisco」において，Sun が支援する Java 関係のオープンソースソフトウェアの Web サイトである「java.net」にてオープンソース化することが発表されました．

それ以来1年ちょっとになりますが，LG3D のコミュニティは java.net で最も活発なものの 1 つとなるまでに成長しました．

また，javadesktop.org にはデスクトップ関連 Java 技術のディスカッションフォーラムがあり，LG3Dのフォーラム (図 1) は「Swing & AWT」をも大差で引き離し，一番活発です．

 

コミュニティ参加の楽しみとは

写真 1 貢献者スポットライトでとりあげられたメンバとプロジェクトオーナ

活発なコミュニティメンバを紹介した「貢献者スポットライト」の Web ページを見るとわかるように，実にさまざまな国からメンバが集まっています (写真 1)．

LG3D コミュニティに参加する理由は人によってさまざまかと思いますが，「Cool な技術を世界各国の人と一緒に開発していけることの喜び」という
声をよく聞きます．この楽しみを読者の皆さんも経験してみてはいかがでしょうか．

 

コミュニティに参加するには

コミュニティに参加する方法は，フォーラムでの議論への参加から，実際に LG3D アプリケーションを開発するまで幅広くあります．

LG3D 日本語フォーラム

ここでは手始めに日本語の LG3D フォーラムを覗いてみましょう．先ほど紹介した javadesktop.org のフォーラムに行くと，中ほどに「Project Looking
Glass 3D」があります．これをクリックすると，3 つの LG3D フォーラムがリストアップされます．ここで，日本語で LG3D 全般に関する議論をするためのフォーラム「Interest Japanese」をクリックすると，日本語フォーラムに辿り着きます．

日本語フォーラムは，活発なときとそうでないときの差がかなりあるのですが，LG3D に関連するイベント前などは，かなりの量の投稿が見られます．

LG3D 英語フォーラム

LG3D に関する多くの議論は，英語フォーラムで行われています．先ほどの 3 つの LG3D フォーラムの中で一番上の「Interest」を選択すると辿り着き
ます．いろいろな国の人たちがさまざまな意見を交換している様子がわかると思います．

よく見てみると，議論をしている人たちのほとんどが，英語が母国語でない人たちです．多少間違った英語でも意味さえ通じれば誰も気にしないので，もし気になる話題があったら，臆することなく返信を投稿してみてください．

● javadesktop.org アカウントの作成

投稿は匿名（Guest ユーザ）でもできますが，javadesktop.org のアカウントを作って，名前 (アカウント名) 付きで投稿されることをお勧めします．名
前を他のコミュニティメンバに覚えてもらうことが，コミュニティ参加への第一歩と言えるでしょう．

アカウントを作るためには，図 1のページの右上の緑の矢印，もしくはその横にある「Login」の文字をクリックしてください．次に現れる「User Login」のページ下部にある「Create Account」ボタンをクリックすると，アカウント作成ページに行きます．その後は画面の指示に従ってください．

ML を介してのフォーラムへの参加

フォーラムの議論には，ML を介しても参加できます．https://lg3d.dev.java.net/ のサイドバーにある「プロジェクトツール」のセクションにある「メーリングリスト」をクリックすると，ML の一覧ページが現れます．「interest_ja」が上述の日本語フォーラム，「interest」が英語フォーラムにリンクしています．

 

LG3D インキュベータ・プログラムとは

「インキュベータ（incubator）」とは卵を孵化させる「孵卵器」のことで，LG3D インキュベータ・プロジェクトは，まさしく LG3D アプリケーションが育つ場所です．誰でも気軽にアプリ作成を開始し，それを早い段階からコミュニティと共有できる場所として設けられました．

本特集を読んで，何かおもしろいLG3Dアプリケーションのアイデアが思い浮かんだら，インキュベータに登録して開発を進めてみてはいかがでしょうか．登録の手順は「ProjectLooking Glassへの貢献」というタイトルのWeb ページに詳しいので参照してください．

 

コミュニティメンバの声を紹介

LG3D では日本人貢献者が大活躍しています．今年の「2005 JavaOne Conference」でDuke's Choice Award を受賞したグループもいます．そんな皆さんに，LG3D や LG3D アプリケーションに対する想いを語っていただきました．メンバからの生の声をお楽しみください．

LG3D との出会い

写真 安藤 幸夫氏

筆者と LG3D との本格的な関わりは，それまで Windows (JDK 5.0) という最新環境でしか動作しなかった LG3D を MacOS X 10.2 (Jaguar) で動かそうとしたところから始まります．その当時の MacOS X では JDK のバージョンも古く，LG3D の基礎となっている Java3D のバージョンも古い環境でした．

果たして LG3D が動作するのか，とても疑問でした．右往左往しながら LG3D コミュニティの助けを借り，MacOS X 上で LG3D が動作したときは感動の一瞬でした．今までさんざん痛い目に遭ってきた Java の "Write Once, Run Anywhere" を再び信じた瞬間でもあったのです．

 

今までの GUI

まず，LG3D 以外の 3D GUI について見ていきたいと思います． 代表的なものとして，SphereXP，3DNA，3DOSX があります．また，Photoshop の GUI を 3 次元に配置するというアイデアもあります．ただ，どれも現時点ではエンターテイメント要素が強く，まだまだ現在のデスクトップ環境をすっかりと置き換えるほどの実用性には到達していないように感じられます．

 

LG3D のめざすもの

LG3D を初めて見た人がよく抱く疑問は，「デスクトップが 3 次元になって使いやすいの？」というシンプルでありながら，本質を捕えたものです．

LG3D が目指しているのは，無意味な 3 次元化ではありません．次世代のデスクトップ環境として，UNIX のコマンドラインの使いやすさと，GUI の使いやすさを併せ持った 2.5 次元的な理想環境を目指しているのです．理想とするのは，習熟した人はより高速に快適に使えるように，そして初めての人にも優しく簡単に使えるような PC 環境です．

キラーアプリとなるもの

そこで，LG3D が数多くの人に使われるようになるための決め手は何でしょうか? LG3D を爆発的に浸透させるためのキラーアプリは，間違いなく File Explorer だと考えます．Macintosh で言うところの Finder，Windows で言うところのエクスプローラです．

普通に PC を使って行う作業の中で，ファイル操作が占める割合は大変多いものです．そして，ファイル操作がスムーズにできるかできないかで使い心地が圧倒的に違うのも事実です．どんなに速い PC でもファイル操作にモタモタするととても使いづらいでしょう．なぜ保存したファイルが見当たらなくなったり，ファイル指定するのに何クリックもしなければいけないのでしょうか？

UNIX/Linux のコマンド操作のほうが高速に操作できる達人も多いでしょう．しかし，PC で扱うファイルの数やサイズは指数的に増加してきており，その環境の中では，旧来の UNIX コマンドが最大限の効力を発揮できない場面も多々出てきているのです．

理想の File Explorer

このような状況を打開するインターフェースとはどのようなものなのでしょうか．筆者の頭の中にある，思い描いた概念だけでもいくつか羅列しておきましょう (表 1)．

PC のパワー，グラフィックスの描画性能に限界があった時代は，まず PC 中心にアプリケーションが構築されていました．

これからは，人の考え方や，動作，行動を中心に，人を主軸にしたアプリケーション環境が整ってくることは間違いないでしょう．

LG3D のFile Explorer への期待

LG3D は見た目の派手さではなく，実用的な使いやすさを重視しています．従って，File Explorer にはとくに力が入れられるはずです．表に挙げたしくみを持つものかもしれませんし，あるいはまったく新しい概念を持つものかもしれません．

LG3D が作り出す未来のユーザインターフェースは，『鏡の国のアリス注5』に敬意を称して，鏡の国の登場人物 “Dum” と命名されるのでしょうか？

最後に，WWW の黎明期に Web の世界に 3 次元をもたらそうとしていた面々が，「なぜ 2 次元デスクトップでも使えるのに，わざわざ 3 次元の世界に向かうのか?」という問いに対する言葉が思い出されます．この言葉にすべてが集約されているのではないでしょうか．「Because the World is not flat. (なぜなら，僕らが住んでいる世界は平らじゃないのだから．)」

はじめに

写真 風間 一洋氏

ある日曜の深夜に川原さんからメールが届いたときに，ふと「いやな予感」がしたのですが，それがこの原稿の依頼でした（笑）．最初は断ろうとしたのですが，「枯れ木も山のにぎわい」だという説得 (?) に負けてしまいました．真面目な話や技術的な話は他の人達が書くそうですから，筆者は与太話を書きましょう．

3D でユーザインターフェースが変わるのか

筆者は研究所に勤めていたおかげで，さまざまな計算機を見る貴重な経験ができました．たとえば，昔は SGI 社のマシンが印象的で，6 自由度デバイス注1を付けて 3D オブジェクトをリアルタイムにグリグリ動かされたら誰でも一発で恋しちゃいそうなくらいでした注2．

しかし，このマシン上で動く，手間もお金もかかって派手な 3D プログラムに驚きながらも，「3D は動画の作成やデータ可視化に役立っても，計算機と人間の関係を変えることは難しいのではないか?」という疑問を持っていました．

 

注 1 3 次元空間内での物体の位置や姿勢を完全に制御するために必要な，x，y，z 軸に関する位置情報 (3 自由度) と回転情報 (3 自由度)，
合わせて 6 自由度のそれぞれに対して，値を指定できるデバイスです．英語では 6DOF (Degree Of Freedom) Device と呼ばれています．

注 2 それが今では同じことが普通の PC でできるのですから，技術の進歩はすごいものです．

 

LG3D が目指す 3D インターフェース

オープンソース化された LG3D に協力することになって，アプリケーションだけでなくデスクトップまでも 3D 化する意義はあるのかと質問に対して，川原さんが「今までの失敗の原因は仮想現実へのこだわりにあった」と答えたときに，その疑問が解消された気がしたのです．

3D デスクトップの意義の 1 つは，「世の中にあり得ないもの」を創り出すことにあると思います．たとえば，デスクトップ上でファイルを探したり，それをゴミ箱に捨てるために，マウスを延々と動かすのは面倒くさいと感じるはずです．そんなとき，ふとデスクトップ上のオブジェクトに魔法をかけたくなりませんか? 物語の中の魔法使いが魔法で家の中の物を操るように，鼻をぴくぴく…は無理ですが (笑)，手をちょっと動かすだけで計算機の中の情報を操る… そんな魔法を，デスクトップの 3D 化と検索技術との組み合わせで実現できたらなあと思っています．

 

オープンソースプロジェクトである利点

現在デスクトップのユーザインターフェースに関しては，皆さまもご存知の Apple Computer の Mac OS X と Microsoft の Windows が普及していて，実際に優れています．しかし，これらはプロプライエタリなソフトウェアですから，勝手に作り変えたりすることはできません．

3D を利用した新しいユーザインターフェースに関するアイデアを実装して試すためには，どうしてもソースコードが公開されている必要があります．そのため，LG3D がオープンソース化されたことは，より多くの開発者を引き付け，さまざまなアイデアが生み出される可能性があります．とくに LG3D は，デスクトップ環境が 1 つのアプリケーションとして動作するモードを提供していますので，この種の試行錯誤はやりやすいと言えるでしょう．

Mac OS X 10.4 "Tiger" を使って一番びっくりしたのは，目玉である Dashboard で，ウィンドウの裏側を設定に使うという LG3D と同じアイデアが使われていたことです．これからも，プロプライエタリなデスクトップ環境にさまざまな影響を与えていくことでしょう．

 

オープンソースプロジェクトであるがゆえの課題

ただし，オープンソースプロジェクトであるがゆえに生じる問題もあります．ユーザインターフェースは，技術的な側面以外にも，認知心理学やデザインアートなどさまざまな側面を持っています．また，新しい概念を生み出すためには，研究と開発，そしてアートと技術の間にプロジェクトを位置させる必要があるでしょう．

このためには，認知心理学者やデザイナなどの技術者以外の協力が不可欠です．その際，バックグラウンドがまったく異なるこれらの人達との間での意思疎通をうまく行う必要がありますが，このようなことは今までオープンソース活動ではほとんど行われてきていません．現在はまだ LG3D の基礎部分を構築している段階ですが，今後筆者らはこのような新しいオープンソース活動のあり方に挑戦していかなければならないでしょう．

はじめに

図 1 Cosmo Scheduler D 動作画面

筆者達の開発しているCosmo Scheduler D (図 1) は，LG3D 上で動作する新しい観点を持
った 3 次元スケジュール帳です．「2005 JavaOne Conference」で Duke's Choice Award を受賞したアプリケーションですが，開発に至るまでにはちょっとした経緯がありました．

 

最初は 3D GUI の開発を目指す

筆者ら学生 3 人が 3D GUI について考え始めたのは，2003 年の始めごろでしょうか．九州工業大学の卒業研究として 3D GUI を考案しようというところから始まります．

筆者らの所属していた小出研究室は並列分散処理の研究を専門としており，まったく GUI とは関係がありませんでした．しかし，小出助教授は前々から 3D GUI の構想を持っておられたようで注1，ひとつやってみないかと話が持ちかけられ，私達は 3D GUI の研究を始めることになったのです．つまり，3D GUI の研究はまったくのイレギュラーだったのです．

 

注 1 2000 年夏のプログラミング・シンポジウムで3 次元デスクトップのデモムービーを上映しました．

 

当初の基本方針

何時間にもおよぶ会議の結果，3D GUI が普及するには「ただ既存のものを 3D 表示するだけでは駄目で，3D デスクトップならではの斬新なワークスタイルを考案する必要がある」という意見で一致しました．

いよいよ実際に 3D GUI の開発に着手するわけですが，3 人それぞれ別のコンセプトで作ってみよう，ということになりました．

  • ウィンドウの 3D 配置，その状態でのアプリケーションの動作検証
  • 3D デスクトップ空間での違和感のない基本操作 (カメラ移動，ピッキング) の検証
  • 3D オブジェクトを掴んでつなげてぶつけるだけで仕事をこなすワークスタイルの検証

 

LG3D に遭遇

3D メーラ?

写真 小田原 大氏

3D の要素を取り込むことにより，大きな変革をもたらすことができるアプリケーションは，プレゼンテーションソフトなどイメージをおもに扱っているものだと考えられます．

その点，メーラはというとイメージでなくテキストをおもに扱っています．それにも関わらずなぜ 3D メーラを開発するのか―そこには，20 年間進化していない 2D デスクトップと同様，進化してない 2D メーラの存在があるからです．

 

どんな殻を破るのか

図 1 3D メーラ「BlackGoat」の動作画面

「BlackGoat注1 (図 1)」のコンセプトは「User-Active (積極的)」と「Effort-Less (楽)」です．つまり，積極的に楽をするということです．ここでは BlackGoat が破る殻の 1 つを紹介します．それは，メールを整理する，という殻です．

2D メーラでメールを整理する場合，種類ごとにフォルダ分けを行うか，大事なメールにフラグ付けを行うぐらいです．

フォルダ分けを行った場合，確かに各フォルダは種類ごとに分かれています．しかし，同じフォルダ内に，件名は異なるけれど内容は関係のあるメールがある場合はどうでしょうか．後で見直す場合，関係のあるメールを逐次探し出さなければならなくなるでしょう．

またフラグの場合，メールのどの部分が重要なのかについては何も指し示すことができないので，結局メール本文すべてをもう一度見直さなければなりません．このように 2D メーラでは十分にメールを整理できるとは言えないのです．

 

注 1 BlackGoat は LG3D インキュベータ・プロジェクトに登録されており，LG3D に付属しています．

 

どうやって殻を破るのか

LG3D の深さ表現とイベントを用います．深さを用いて 3D 表現できるため，フォルダ内のメールを「1 枚の紙」，重要フラグを「付箋」として表現します．さらに関係のあると思うメールを「クリップ」でまとめるのです．こうすることにより，フォルダ中にどういったメールがあるかを視覚的に判断できるようになります．また，イベントを用いて「落書き」を表現できるため，具体的にメールのどの部分が重要なのかメモをとることができるようになります．

大事なメールに付箋を貼り，関係のあるメールをクリップでまとめる．そして，必要があればメモを加える．そう，ふだん私たちが書類を整理するために行っていることをデスクトップ上で表現することにより，メールを整理する殻を破るのです．

 

今後の展開

BlackGoat は，Google が提供している，学生を対象にオープンソース開発に奨学金が支払われるプログラム「Summer Of Code」に登録されました．この機会を活かし，3D の可能性を追求しながら BlackGoat をさらに進化させていきます．

Project Looking Glass 参加の手引き

どこが凄いの?

写真 えんどう やすゆき氏

LG3D はどこがすごいのでしょうか? それは実際に動いているところを見ないとわからないでしょう．筆者も実際に動作しているデモを見たときは想像以上の衝撃を受けました．何よりすごいのはデスクトップそのものが 3 次元空間になってい
る点でしょう．これにより2次元のウィンドウ枠の呪縛から開放され，3 次元の物体を直接デスクトップに置けるようになったのです．

Web ブラウザのウィンドウが裏返ったり，カメラがパンするようにデスクトップの背景が横移動したりするのを見て興奮を隠せませんでした．なぜなら，3 次元に広がったデスクトップでは，今まで使えなかった z 軸 (奥行き) 方向が使え，従来のアプリケーションにはない使い勝手が実現できるのではないかと考えたからです．

これを作ったのが日本人のエンジニアであるということにも興奮しました．全世界の人々が参加するオープンソース開発プロジェクトを主導する川原氏を心から応援したいと思いました．

 

どう楽しいの?

図 1 Ls3D 動作画面

最も楽しいのは仲間との交流です．筆者が LG3D に興味を持ったとき，すでに発足していた LG3D の日本語コミュニティで翻訳者を募集していました．自分自身がその文書を必要としていたこともあって，筆者は翻訳者に応募しました．

筆者の英語力は謙遜ではなく決して高いものではありません．今考えると無謀な試みでしたが，コミュニティ参加者からいただいたていねいなコメントのおかげでいくつかの翻訳を公開できるようになりました．

また今年の 3 月に六本木で行われた LG3D BoF では，自作のプログラム「Ls3D」をデモする機会にも恵まれました．UNIX の ls コマンドと似たファイル名のリストを 3 次元空間に表示するだけのプログラムですが，z 軸 (奥行き) の使い方の一例を示しました (図 1)．

幸い英語コミュニティでも話題になり，LG3D のアプリケーションを集める「LG3D インキュベータ・プロジェクト」に採用されることになりました．

当初は筆者が描いた貧弱なアイコンが使われていたのですが，デザインセンスある参加者から立派なアイコンを提供されて大変感激しました．

 

プログラミングのコツは?

Java プログラミングをしたことがあれば難しくありません．AWT や Swing を自然な形で 3 次元に拡張した API は，Java プログラマにとって理解しやすいものです．本特集 2 章の櫻庭さんによる平易なチュートリアルを参考にしてください．

3 次元グラフィックスは奥が深いものです．LG3D の背景にある 3 次元グラフィックス理論の一端を知るには，まず Java の標準的な 3 次元グラフィックスAPIである Java 3D を知るのが良いでしょう．そのための 1 つの手引きとして，筆者が 2003 年に出版した『Java 3D プログラミング・バイブル』 (ISBN : 4-8163-3430-0) という書籍をご紹介しておきます．

 

理想の3D エンジンをめざして

(株) エイタロウソフト代表取締役社長 CEO 西島栄太郎

LG3D との出会い

写真 西島栄太郎氏 (中央) とエイタロウソフトの方々

(株) エイタロウソフトは，3D モデル表示エンジンの提供を通して，LG3D のオープンソースコミュニティ活動に参加しています．

当社はもともとは Java Applet で動作する Web の 3D エンジンの開発を行っていました注1．3D エンジンを提供する人間として常日頃考えている重要なテーマがあります．それは，「いかに 3D を活用するコンテンツを生み出すか」です．

単純に 3D と言ってしまうと，一昔前のイメージでは商品やロゴをくるくると回す程度でした．最初はもの珍しいのですが，すぐに飽きてしまい，つまらないという印象を持っていました．

3D を活用するために 3D らしさを強調することばかり考えていた筆者は非常に感銘を受け，あえて 3D を誇張することなく本当にユーザの使い易さを考えれば良いのだと思い知らされました．

LG3D での活動

図 1 MXW3D で作成された Duke の 3D マスコット

そこで私も LG3D の新しい可能性として，もっと手軽に 3D を利用できる環境を提供したいと考えるようになりました．調べてみると LG3D には数多くの 3D モデルが利用できるように，あらかじめモデルローダのしくみが用意されていることがわかりました．さすがに次世代の 3D デスクトップ環境だけあり，非常に進んだ設計に感動します．

筆者が目指したのは，すでに存在している単純な 3D モデルの読み込みを超えた，ゲーム開発にも使用されているボーン構造注2による豊かで自然なアニメーションや，金属の光沢や映りこみを再現する多彩な表現を簡単に使用することができる 3D エンジン「MXW3D (Mobile eXtension Web3D)」です (図 1)．

一見するとこのような機能は “お楽しみ” の要素だけが強く感じられますが，実際にはオフィスツールのアシスタントに 3D マスコットを使用したり，ビジュアルデザイナのデザインした豊かなインターフェースを簡単に再現することができるなど，ビジネスにも有効に利用できます．

LG3D が次世代のデスクトップとして最初から 3D の動作を想定されているため，こうして作られた 3D モデルを他のウィンドウと同様に整列したり自在にユーザが操作できることには驚かされます．

 

注 2 実際の動物や人体と同様に骨 (ボーン) を用いて 3D モデルを動かす構造のことです．

 

最後に

今回の 2005 JavaOne Conference で実際に 3D マスコットのデモを提供させて頂きましたが，ようやく 3D モデリングツールからコンバートして簡単に利用できる環境を準備することができました．これを機に次世代の 3D デスクトップならではの独創的なアプリケーションが生まれてくるのを楽しみにしています．
