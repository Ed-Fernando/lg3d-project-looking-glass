Contributing to Project Wonderland and Project Looking Glass

The source code for Project Looking Glass is available to anyone to use according to the licensing terms. Development of the projects happens in public and is open to individuals and companies. Everyone benefits by having better projects when the fixes or enhancements are implemented. See the JavaDesktop Community Governance Guidelines for general guidelines on participating in JavaDesktop community projects, such as Project Looking Glass.

Joint Copyright Assignment

All contributers of source code, object code, documentation, or other materials must complete the Joint Copyright Assignment form before any contributions can be included in the project. Print, complete, sign, scan and email the Joint Copyright Assignment form (JCA FAQ) to: lg3d-jca-submit@sun.com.

Individuals and companies who have signed and submitted Joint Copyright Assignment forms are listed here.

How You Can Contribute

Here are some of the ways you can contribute to Project Looking Glass:

  • Try out the binary release
    You can download the binary release of Project Looking Glass from the Documents & files section of the lg3d-core project.

  • Participate on Forums and Mailing Lists
    The Project Looking Glass forums and of Project Wonderland forums mailing lists provide a way for people using the software to benefit from each other's expertise. To really be useful, these lists need your input! Please don't be afraid to post: help out others when you can, post your questions, comments and ideas as you see fit. See the Staying Connected section on the lg3d-core page for more information.

  • File a Bug Report
    Bug reports are very helpful to developers, and a great way to contribute to a project! Please use the Issue Tracker in the lg3d-core project to file bugs (defects). Remember to take a little time when filing a bug report; make sure you can reproduce the problem, and be very explicit in your description of the problem.

  • File an Enhancement Request
    If there's something you'd like to see in a project and don't, or something you can imagine improving, let us know by filing an enhancement request. Use the Issue Tracker and file an "Enhancement" rather than a "Defect".

  • Contributing Code to lg3d-core
    As mentioned above, all of the source code is available, and if there's a feature you'd like to see, or a bug you'd like to see fixed, you have the power to make that happen. In order to directly contribute source code, you need to have a Developer role in the project. However, before signing up for it, please consider becoming a project Observer, downloading the source and familiarizing yourself with API internals. Don't forget to submit a Joint Copyright Assignment form if this is your first contribution.

    The typical code update procedure we are practicing looks like the following:

    1. File a bug or request for enhancement using the Issue Tracker.
    2. Discuss fixes using the "Additional comments" section of the Issue page. It is a good idea to attach a patch to the Issue page. If you'd like to discuss the issue with wider audience, please use the Project Looking Glass forums
    3. Once we (at least including one of the project owners) agree on the changes, commit those to the master CVS.

    Although we sometimes don't follow these steps exactly, we'd suggest you to follow them until you get accustomed to the codebase and making changes. You may want to subscribe to the cvs@lg3d-core.dev.java.net and the issues@lg3d-core.dev.java.net aliases from this page to see how things are going.

  • Contributing LG3D Applications at the Incubator
    Lg3d-incubator is the place for you to share your cool LG3D applications! You can grow it together with the community if you wish. If you want to start a new LG3D application development, you are encouraged to post your idea at the Project Looking Glass forums. You'll get many informative feedback. In order to upload your code and share it with the community, you need to submit a Joint Copyright Assignment form if you haven't done so yet. Then, you'll be granted a Developer role in the incubator project.

  • Contributing Web Contents
    We are looking for some help for enriching the web contents. In order to directly contribute contents, you need to have a Content Developer (or Developer) role in the project. Before applying for the role, please don't forget to submit a Joint Copyright Assignment.

  • Translation
    You can also help by assisting in the translation of these pages as well as documentation. Work is being done on translating these pages to several languages as you see on the lg3d-core front page. We would like to extend translation to include other languages so that we can grow the world wide LG3D community. In order to do this you need to have a Content Developer (or Developer) role. The procedure is the same as signing up for contribution of other code.

    • If you'd like to start a new translation project to a language other than the ones listed as a on-going sub-project , please first express your interest at the Project Looking Glass forums.
    • If you are interested in joining the on-going work, check out the list here and make a contact with the lead, or express your interest at the forum.

    Please reference to the Translation File Style Guide for detailed guide line for the file format and other nice ideas to make your translation work easier.
