1. Požadavky na platformu

Před tím než začnete nastavovat vývojové prostředí pro Projekt Looking Glass, zkontrolujte prosím nejprve, zda váš systém splňuje následující minimální požadavky:

Komponenta Platforma Linux
Operační systém Většina moderních linuxových distribucí by měla fungovat, avšak setkali jsme se s určitými problémy při běhu na instalacích javy založených na portage. Předpokládáme následování této příručky pro nainstalování všech komponent ručně namísto použití portage.

RAM 512MB
Grafická karta*
 3D akcelerovaná grafická karta s ovladačem podporujícím OpenGL verze 1.2 nebo vyšší.
DŮLEŽITÉ! Je vyžadována 24 bitová hloubka barev! Podívejte na instrukce uvedené níže, které uvádějí jak provést tuto konfiguraci.

2. Nastavení prostředí pro Projekt Looking Glass

Ještě než budete moci spustit Projekt Looking Glass, musíte nainstalovat několik Java komponent. Nejjednodušší cestou je stáhnout všechny komponenty a poté je nainstalovat v níže uvedeném pořadí.

Stáhněte komponenty

Poznámka: Instalační instrukce předpokládají, že všechny komponenty stáhnete do adresáře /tmp.

  1. Java 2 SDK, Standard Edition 1.5.0 nebo pozdější.
    http://java.sun.com/j2se/1.5.0/download.jsp

    Poznámka: Níže uvedené instrukce pro instalaci předpokládají stažení RPM verze JDK.

  2. Java 3D SDK 1.3.2 nebo pozdější
    https://j3d-core.dev.java.net/servlets/ProjectDocumentList

    Poznámka: Přejděte do podadresáře j3d-core/Release_Builds/1.3.2 a stáhněte java3d-1_3_2-linux-i586.zip.



  3. Java Advanced Imaging API (JAI) 1.1.2 JDK
    http://java.sun.com/products/java-media/jai/downloads/download-1_1_2.html

    Poznámka: Zvolte JDK(TM) Install: Bundle for installation in a JDK Download. Po přečtení a odsouhlasení Software License Agreement (licence) vyberte položku Linux JDK Install (jai-1_1_2-lib-linux-i586-jdk.bin).

Nainstalujte komponenty

Poté co jste stáhli všechny komponenty, nainstalujte je podle následujících instrukcí.

  1. Nejprve se přepněte na uživatele root:

    % su root
    Password:     vložte heslo uživatele root
    #

  2. Nainstalujte Java 2 SDK:

    # cd /tmp
    # /bin/bash jdk-1_5_0_02-linux-i586-rpm.bin
    # rm jdk-1_5_0_02-linux-i586.rpm
    # rm jdk-1_5_0_02-linux-i586-rpm.bin

  3. Nainstalujte Java 3D SDK:

    # cd /tmp
    # unzip java3d-1_3_2-linux-i586.zip
    # cd /usr/java/jdk1.5.0/jre
    # unzip /tmp/java3d-1_3_2-linux-i586/j3d-132-linux-x86.zip
    # cd /tmp
    # rm -r java3d-1_3_2-linux-i586/

  4. Nainstalujte Java Advanced Imaging API:

    # cd /usr/java/jdk1.5.0
    # /bin/bash /tmp/jai-1_1_2-lib-linux-i586-jdk.bin

  5. Opusťte root shell:

    # exit
    %

  6. Nyní je třeba nastavit několik proměnných prostředí v profilu vašeho shellu. Zde je příklad pro bash(1):

    JAVA_HOME=/usr/java/jdk1.5.0
    PATH=$JAVA_HOME/bin:$PATH

3. Nainstalování Vývojářské verze Projektu Looking Glass

Zaregistrujte se na webu java.net

Pokud již nejste zaregistrováni na java.net, pak si budete muset založit účet:

  1. Přejděte na http://www.java.net a klikněte na odkaz Register v horní části stránky.
  2. Zadejte zvolené uživatelské jméno a emailovou adresu a stiskněte tlačítko [Register].
  3. Přijde vám email s instrukcemi pro nastavení hesla vašeho účtu. Poté co vložíte heslo a odsouhlasíte podmínky registrace na webu java.net, budete přihlášeni.

Stáhněte instalační balíček Projektu Looking Glass

  1. Jděte na stránky https://lg3d-core.dev.java.net a klikněte na Documents & files v sekci Project tools na postranní liště.
  2. Přejděte do lg3d-core/stable_builds ve stromu v pravé části panelu. Zobrazí se vám tabulka se seznamem stabilních buildů.
  3. Klikněte na odkaz pojmenovaný Release_x.y.z (kde x.y.z je verze buildu, například 0.6.1) a zvolte adresář, kam bude soubor stáhnut, jako třeba /tmp
  4. Změňte aktuální adresář na adresář kam budete chtít Projekt Looking Glass nainstalovat a rozbalte soubory:

    % cd /home/username
    % tar -zxf /tmp/lg3d-x.y.z.tar.gz

    Poznámka: Nahraďte x.y.z verzí buildu, který jste stáhli (např. 0.6.1).

    Soubory budou rozbaleny do adresáře lg3d umístěného v aktuálním adresáři.
Aktuální verze LG3D je verze 0.6.1. Než začnete tuto verzi používat, přečtěte si i Poznámky k verzi.

Nastavte barevnou hloubku vašeho displeje

Vývojářská verze Projektu Looking Glass vyžaduje nejméně 24 bitovou barevnou hloubku. Mnoho systémů je však nakonfigurováno se 16-ti bitovou barevnou hloubkou a Vývojářská verze Projektu Looking Glass nebude v této barevné hloubce pracovat správně. Aktuální nastavení barev vašeho displeje můžete zjistit spuštěním následujícího příkazu:

% xdpyinfo | grep "depth of root"
depth of root window:   24 planes

Pokud se zobrazí hodnota menší než 24, musíte změnit nastavení barevné hloubky. Proces změny nastavení hloubky barev se liší pro různé platformy. Podívejte se proto na konfigurační instrukce pro vaši platformu, uvedené níže:

Java Desktop System 1 a 2, a SUSE8.1:

  1. Z desktopu spusťte příkaz sax2:

    % /usr/X11R6/bin/sax2
    SaX: root Password:     vložte heslo uživatele root

  2. V aplikaci SaX2 klikněte na Color and Resolution a poté na tlačítko [Properties...] pro aktuálně použitý desktop.
  3. V panelu Color selection... klikněte do menu a vyberte 16.7 Mio. [24 Bit].
  4. Na záložce Resolution(s) for 16.7 Mio. [24 Bit] colors vyberte preferované rozlišení vašeho desktopu a stiskněte [Ok].
  5. Klikněte na tlačítko [Finish] a poté na [Finalize>>].
  6. Otevře se dialog SaX2 final steps.... Stiskněte tlačítko [Test...] a otestujte tak nové rozlišení.
  7. Měla by se objevit modrá testovací obrazovka. Klikněte na tlačítko [Save], jestliže se nastavení displeje jeví stabilní. Po návratu do okna SaX2 klikněte na [Ok] a poté na [Yes] pro ukončení SaX2.
  8. Pokud se test nezobrazí správně, dojde po třiceti sekundách k jeho ukončení a obnoví se předchozí nastavení desktopu. Měli by jste zkusit alternativní rozlišení při barevné hloubce 24 bitů a najít takové které na vašem systému bude pracovat správně.
  9. Jste-li s nastavením spokojeni, odhlašte se a znovu se přihlašte do desktopu. Pomocí uvedeného příkazu xdpyinfo pak zkontrolujte, že barevná hloubka je skutečně 24 bitů nebo více.

RedHat Fedora 2:

  1. Nejprve se přepněte na uživatele root:

    % su root
    Password:     vložte heslo uživatele root
    #

  2. Otevřete v editoru soubor: /etc/X11/xorg.conf
  3. Najděte část souboru začínající textem: Section "Screen"
  4. V této části najděte položku: Identifier "Screen0"
    Změňte vlastnosti DefaultDepth a DefaultFbBpp na:

    DefaultDepth 24
    DefaultFbBpp 32

  5. Uložte soubor, odhlašte se a znovu se přihlašte do desktopu a pomocí uvedeného příkazu xdpyinfo zkontrolujte, že nyní je barevná hloubka 24 bitů nebo více.

RedHat 8, RedHat 9 a ostatní linuxové platformy:

Mnoho linuxových platforem používá pro nastavení vlastností displeje soubor XF86Config. Změna barevné hloubky se v takovém případě provede následovně:

  1. Nejprve se přepněte na uživatele root:

    % su root
    Password:     vložte heslo uživatele root
    #

  2. Otevřete v editoru soubor: /etc/X11/XF86Config
  3. Najděte část souboru začínající textem Section "Screen" a změňte vlastnost DefaultDepth na 24:

    DefaultDepth     24

  4. Uvnitř této části najděte podčást označenou "Display":

    SubSection "Display"

    a změňte vlastnost Depth na:

    Depth     24

  5. Uložte soubor, odhlašte se a znovu se přihlašte do desktopu a pomocí uvedeného příkazu xdpyinfo zkontrolujte, že nyní je barevná hloubka 24 bitů nebo více.

4. Spouštění Projektu Looking Glass

Existují dva způsoby jak spouštět Projekt Looking Glass; uvnitř okna existujícího desktopového prostředí a nebo jako celoobrazovkový desktop. Pouze celoobrazovkový režim podporuje klíčovou vlastnost Projektu Looking Glass a to integraci nativních nemodifikovaných X11 aplikací. Nejjednodušší způsob jak ověřit, že instalace Projektu Looking Glass je provedena správně, je spustit demo v existujícím desktopu. Ve chvíli kdy si ověříte správnou funkci, můžete nakonfigurovat váš systém pro spouštění Projektu Looking Glass v celoobrazovkovém režimu. Upozorněme, že spouštění Projektu Looking Glass v okně desktopu je výhodné při vývoji a testování 3D aplikací a rozšířování manažeru oken (Window Manager, Scene Manager).

Spouštění Projektu Looking Glass v existujím desktopovém prostředí

V tomto módu běží Projekt Looking Glass v okně vašeho desktopu. Jde o nejjednodušší způsob spouštění Projektu Looking Glass, ale umožňuje používat pouze 3D aplikace Projektu Looking Glass, jako je například databáze CD. X11 aplikace, jako je například xterm se objeví v X session ze které byl Projekt Looking Glass spuštěn.

  1. Přihlašte se pod svým účtem a změňte adresář na ten, kam jste nainstalovali Projekt Looking Glass:

    % cd /home/username/lg3d/bin

  2. Spusťte session Projektu Looking Glass:

    % ./lg3d-dev

Pokud jste úspěšně nainstalovali Projekt Looking Glass, měli by ste v okně na vaší pracovní ploše vidět známý desktop Projektu:

Kliknutí na ikonu CD v panelu otevře katalog CD, 3D demo aplikaci. Abyste viděli aplikaci v akci, pohněte myší nad balíček CD a klikejte na jejich obrázky. Pro další informace o operacích klikněte na ikonu nalevo od Prohlížeče CD na liště úloh. Tím zobrazíte panel, který shrnuje informace o operacích podporovaných demo aplikací. Všimněte si i ikony počítače úplně vlevo, jež spouští xterm. Takto je demonstrována integrace X11 aplikací a je podporavána pouze v celoobrazovkovém režimu. Nastavení tohoto režimu uvádí následující sekce.

Spouštění Projektu Looking Glass v celoobrazovkovém režimu

Tento mód umožňuje provozovat Projekt Looking Glass na celou obrazovku a spouštět jakoukoliv aplikaci, včetně nativních X11 alikací. Před spuštěním Projektu Looking Glass musíte nejprve ukončit X session. Tento režim lze použít pouze na linuxových systémech. Časem bude tento mód primární pro použití LG3D a bude podporován jak na Linuxu tak na Solarisu

Poznámka: Na tomto režimu použití se stále ještě intenzivně pracuje. V současnosti takto funguje bezchybně jen několik X11 aplikací. Ještě stále zbývá spousta ladicí práce. Důležité aplikace, jako například mozilla, staroffice, evolution, realplayer a vetšina GNOME aplikací, mají ještě významné problémy. Těmito věcmi se aktivně zabýváme. Prosíme, reportujte každý problém, který najdete.

Instrukce se liší dle platformy, proto postupujte dle instrukcí určených pro vámi používaný systém. Jestliže vaše platforma není uvedena, je pravděpodobné, že postup bude velmi podobný a proto můžete instrukce použít jako vodítko a následně nám dát vědět o nalezených rozdílech!

Sun Java Desktop System/SuSE 8.1:

JDS/SuSE 8.1 používá X Display Manager (xdm) pro správu uživatelských session. Před spuštěním Projektu Looking Glass tedy nejprve musíte ukončit xdm. Protože je tento proces automaticky obnovován, musíte službu vypnout provedením příkazu chkconfig:

  1. Nejprve se přepněte na uživatele root:

    % su root
    Password:     vložte heslo uživatele root
    #

  2. Vypněte xdm službu:

    # /sbin/chkconfig -d xdm
    xdm  0:off   1:off   2:off   3:off   4:off   5:off   6:off

  3. Nyní zavřete všechny desktop aplikace a uložte otevřené soubory. Poté ukončete display manager:

    # pkill gdm-binary

  4. Přejděte do sekce Spouštění desktop session Projektu Looking Glass.

Poznámka: Pro obnovení X Display Manageru, zadejte jako root následující příkaz a poté restartujte systém:

# /sbin/chkconfig -a xdm

RedHat 9:

RedHat používá ke správě uživatelských session GNOME Display Manager (gdm). Není nutné vypínat gdm, stačí systém přepnout do nižšího runlevelu, ve kterém gdm není aktivní:

  1. Nejprve se přepněte na uživatele root:

    % su root
    Password:     vložte heslo uživatele root
    #

  2. Zavřete všechny desktop aplikace a uložte otevřené soubory. Poté přepněte do runlevelu 4:

    # /sbin/init 4

    Vaše X session by měla být ukončena a sytém by se měl vrátit do příkazové řádky.

  3. Nyní přejděte do sekce Spouštění desktop session Projektu Looking Glass.

Poznámka: Pro restartování GNOME Display Manageru, zadejte z konzole uživatele root následující příkaz:

# /sbin/init 5

Fedora Core 3 64-bit:

Pro běh Projektu Looking Glass na celé obrazce v 64-bitové verzi Fedora Core 3 je třeba provést několik specifických úprav. Upozorněme, že k jejich provedení budete potřebovat přístup k 32-bitovému linuxovému systému nebo 32-bitovým X11 modulům pro Linux.

  1. Pro provoz LG3D je nutné zkopírovat obsah aresáře /usr/X11R6/lib/modules ze 32-bitového linuxového systému do stejného adresáře vašeho 64-bitového systému. Před touto akcí zazálohujte obsah adresáře /usr/X11R6/lib/modules ve vašem 64-bitovém systému:

    # cd /usr/X11R6/lib
    # cp -r modules modules.orig

    Nyní použijte příkaz scp pro bezpečené zkopírování souborů z 32-bitového systému na váš 64-bitový systém:

    # scp -r root@32bithost:/usr/X11R6/lib/modules /usr/X11R6/lib/modules

    kde 32bithost je jméno 32-bitového linuxového systému.

  2. Ujistěte se, že v souboru xorg.conf je správně nastavena barevná hloubka na 24, tak jak je tučně vyznačeno níže:

    Section "Screen"
    Identifier "Screen0"
    Device "Videocard0"
    Monitor "Monitor0"
    DefaultDepth 24
    SubSection "Display"
    Viewport 0 0
    Depth 24
    Modes "1680x1050" "1280x800" "1024x768" "800x600" "640x480"
    EndSubSection
    EndSection
  3. Pokračujte následující sekcí pro start celoobrazovkové session.

    [Instrukce v této sekci uvedeny se svolením qgonjona a marva. Podívejte se také na tento diskuzní thread.]

Spouštění desktop session Projektu Looking Glass

Pro start desktop session Projektu Looking Glass z konzole (nikoli z X-session):

  1. Přepněte se na uživatele root a přejděte do adresáře, kde máte nainstalován Projekt Looking Glass:

    # cd /home/username/lg3d/bin

  2. Spusťte desktop session Projektu Looking Glass:

    # ./lg3d-session

    Po několika vteřinách byste měli vidět desktop Projektu Looking Glass. Gratulujeme! Pokud máte problémy se spuštěním desktop session, podporu vám poskytne sekce Řešení problémů.

Prozkoumejte desktop Projektu Looking Glass a začněte snít o nových způsobech použití dektopu v rámci 3D! Chcete-li se účastnit vývoje, přečtěte si Příručku vývojáře Projektu Looking Glass, kde naleznete potřebné informace.

5. Řešení problémů

no core pointer

Toto nastává na RedHat Fedora Core 2 a zřejmě se může týkat i dalších konfigurací. Problém je možné obejít (postup od sgiunchiho) provedením následujícího (jako uživatel root):

# ln -s /etc/X11/xorg.conf /etc/X11/XF86Config-4

Korektním řešením je upgradovat verzi X11, která je součástí lg3d-core. V současnosti jde o Xorg 6.7 pre-release a je třeba ji upgradovat na poslední stabilní verzi Xorg.

Exception in thread "main" java.lang.UnsatisfiedLinkError: /usr/java/jdk1.5.0/jre/lib/i386/libj3dcore-ogl.so:
/usr/java/jdk1.5.0/jre/lib/i386/libj3dcore-ogl.so: symbol glMultiDrawArraysEXT, version LIBGL not defined in file libGL.so.1 with link time reference

Tato chyba nastává s ovladači ATI, které postrádají rozšíření glMultiDrawArraysEXT. Řešením je nahradit Radeon ovladače ovladači z projektu Direct Rendering Open Source (DRI). Instrukce pro instalaci ovladačů DRI obsahuje dokument Instalace ovladačů DRI pro ATI Radeon.

Upozorněme, že upgrade libGL.so na poslední verzi knihovny Mesa tento problém nevyřeší.

pkill: 27863 - Operation not permitted
java.rmi.server.ExportException: Port already in use: 1099; nested exception is:
      java.net.BindException: Address already in use...

Tato chyba indikuje, že proces rmiregistry již beží a lg3d-dev nemohl spustit novou instanci. Typicky nastává pokud byl rmiregistry proces spuštěn uživatelem root a lg3d-dev je spouštěn pod jiným uživatelem. Řešením je ukončit instanci rmiregistry uživatele root. Přepněte na uživatele root a ukončete proces:

% su root
Password: vložte heslo uživatele root
# pkill rmiregistry
# exit

Pro spuštění Projektu Looking Glass nyní znovu proveďte příkaz lg3d-dev jako běžný uživatel.

Fatal Server error: Cannot establish any listening sockets

Při startu Projektu Looking Glass pomocí lg3d-session se zobrazí následující chybová zpráva:

_XSERVTransSocketUNIXCreateListener: ...SocketCreateListener() failed
_XSERVTransMakeAllCOTSServerListeners: server already running

Fatal server error:
Cannot establish any listening sockets - Make sure an X server
isn't already running

Nejprve se ujistěte, že na displeji na kterém spouštíte lg3d-session neběží žádná X11 session. Poté zkontrolujte zda z předchozí session nezůstal soubor /tmp/.X11-unix/X0. Aby byl problém vyřešen, je třeba jej případně smazat a restartovat lg3d-session.

Překreslování obrazovky je velmi pomalé

Jde obvykle o důsledek nesprávně nastavené akcelerace. Nejprve zkontrolujte následující:

  • Spusťte příkaz glxinfo z X11 session a zkontrolujte, že položka direct rendering je nastavena na Yes. Podívejte se rovněž na odkazy na OpenGL, abyste se ujistili, že vaše grafická karta podporuje OpenGL.
    - Pro karty NVIDIA spusťte xdpyinfo a zkontrolujte, existenci položek GLX či NVIDIA-GLX.

    Jestliže tyto položky nevidíte, pravděpodobně nemáte nainstalován správný ovladače nebo není zapnutá podpora 3D. Informace o instalaci a konfiguraci ovladače s podporou 3D/OpenGL byste měli vyhledat na webu výrobce vašeho zařízení.

  • Zkontrolujte, zda log soubor X serveru /var/log/XFree86.0.log neobsahuje chybová hlášení a prověřte jaké ovladače jsou používány.

libGL error: failed to open DRM: operation not permitted

(Tato chyba se může objevit při běhu lg3d-dev).

V případě, že Projekt Looking Glass běží velmi pomalu a to i přesto, že 3D akcelerace je zapnuta, zkontrolujte, zda se v konzoli nezobrazuje následující zpráva:

libGL error: failed to open DRM: Operation not permitted
libGL error: reverting to (slow) indirect rendering

Toto znamená, že váš uživatelský účet nemá oprávnění k použití DRI (Direct Rendering Infrastructure). V některých případech je root jediným uživatelem s přístupem k Direct Rendering. Zdá se, že se jedná pouze o problém systému RedHat a nikoli systémů JDS/SuSE. Pokud se u vašeho nastavení tato chyba projevuje, musíte změnit DRI práva ve vašem souboru XFree86Config či XF86Config takto:

Section "DRI"
Group "video"
Mode 0666
EndSection

Rovněž musíte změnit práva profilu <dri> v souboru /etc/security/console.perms z 0600 na 0666:

<xconsole> 0600 <dri> 0666 root

a pokud používáte kartu NVIDIA, změňte práva zařízení /dev/nvidia* na 0666:

# chmod a+rw /dev/nvidia*

An unexpected exception has been detected in native code outside the VM

Projekt Looking Glass skončí následující výjimkou i v případě, kdy jste se ujistili, že 3D akcelerace je zapnuta:

An unexpected exception has been detected in native code outside the VM.
Unexpected Signal : 11 occurred at PC=0x0
Function=[Unknown.]
Library=(N/A)

Zkontrolujte, zda se v chybové zprávě výjimky nacházejí následující dva záznamy:

<addr-range> r-xp 00000000 03:05 61763 /usr/lib/GL/libGL.so.1.3.mesasoft
<addr-range> rw-p 001b4000 03:05 61763 /usr/lib/GL/libGL.so.1.3.mesasoft

Pokud ano, máte nainstalovány OpenGL knihovny MESA. Použijte balíčkovací nástroj vaší distribuce (např. rpm) k jejich odstranění. Poté přejděte do /usr/lib jako root a zkontrolujte, že následující symbolické linky existují a ukazují na správné OpenGL knihovny. Linky by měly vypadat podobně jako následující (kde x a y jsou čísla):

libGLcore.so.1 -> libGLcore.so.1.x
libGL.so -> libGL.so.x
libGLU.so -> libGLU.so.x.y.z

Pokud tyto linky nejsou správné, přepněte se na uživatele root a vytvořte je. Například pokud máte soubor libGLU.so.1, ale neexistuje link libGLU.so, pak jej vytvořte:

# ln -s /usr/lib/libGLU.so.1 /usr/lib/libGLU.so

Toto zopakujte pro každý chybějící link.

Nejde nahrát ovladač NVIDIA

Existuje několik možných důvodů nesprávné inicializace ovladače:

  • Ovladač NVIDIA je zkompilován pro linux.*bigmem. Pokud má váš systém dva obrazy (image), bigmem a linux*up, použijte první z nich. V opačném případě nahrání ovladače selže.

  • Ovladač NVIDIA musí najít IRQ přiřazené grafické kartě. To by měl nastavit BIOS před bootováním operačního systému, avšak tuto funkci lze u některých počítačů vypnut. Zkuste tedy toto nastavení přepnout zpět a zjistit jestli se tím problém vyřešil.

  • Projděte instalační instrukce k ovladači NVIDIA a poznámky k vydané verzi pro zjištění dalších detailů.
