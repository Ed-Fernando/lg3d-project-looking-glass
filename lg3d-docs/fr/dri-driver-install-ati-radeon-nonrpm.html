Installation du pilote DRI pour les ATI Radeon


Ces instructions expliquent comment installer les pilotes de l'Infrastructure de Rendu Direct (DRI) pour les cartes graphiques ATI Radeon afin d'activer l'extension de rendu direct requise par le Projet Looking Glass.

Instructions d'installation

Ici vous trouverez un guide pour installer les pilotes binaires d'ATI sur une distribution non basée sur rpm comme Slackware. Je me réfererai aux commandes nécessaires avec slackware mais la plupart des étapes peuvent être appliquées sur les autres distributions.

Note : Toutes les versions récentes (> 3.12) des pilotes fglrx contiennent une implémentation incomplète de GL. Donc il se peut que lg3d ne marche pas correctement avec ces pilotes. Si vous rencontrez des problèmes, veuillez essayer les autres instructions installation.

  1. Passez en root:

    % su -
    Password: donnez le mot de passe root
    #

  2. Installez les sources du noyau (kernel sources) depuis le paquet kernel-source de la distrib :

    # installpkg kernel-source-2.4.26-noarch-4.tgz

    Note : Dans la distribution Slackware ce paquet est sur le premier CD.
    /mnt/cdrom/slackware/k/kernel-source-2.4.26-noarch-4.tgz

  3. Mettez à jour la configuration du noyau et les dépendances :

    # cd /usr/src/linux
    # cp /boot/config-ide-2.4.26.config
    # make oldconfig
    # make dep

    Note : Vous n'avez pas à recompiler le noyau.

  4. Téléchargez les derniers pilotes pour ATI Radeon sur le site web d'ATI.

  5. Convertissez le rpm en un fichier tar.gz normal avec :

    # rpm2tgz fglrx-"version".rpm
    # cd /
    # tar zxvf fglrx-"version".tgz
    # cd /lib/modules/fglrx/build_mod/
    # chmod +x ./make.sh && ./make.sh
    # cd .. && chmod +x make_install.sh && ./make_install.sh

  6. Lancez fglrxconfig pour configurer X :

    # fglrxconfig

  7. Sur Slackware 10 ainsi que sur de nombreuses distributions utilisant xorg au lien de XFree86, vous devrez faire un lien symbolique entre XF86Config-4 et xorg.conf :

    # ln -sf /etc/X11/XF86Config-4 /etc/X11/xorg.conf

    Maintenant essayez de lancer X, à moins que vous n'ayez rencontré une erreur dans les étapes précédentes vous devriez avoir une accélération 3D opérationnelle maintenant. Vérifiez le en lançant :

    % glxinfo | grep render

    Vous devriez voir:

    direct rendering: Yes
