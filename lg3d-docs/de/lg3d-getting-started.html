Einstieg in das Projekt Looking Glass (Entwickler Version)

Willkommen beim Project Looking Glass!

Dieses Dokument dient als Anleitung wie du am schnellsten die Entwicklerversion von Looking Glass unter Linux oder Solaris x86 Maschinen zum laufen bekommst. Um den Installation Prozeß erheblich zu vereinfachen, wird das LG3D Mega-bundle zur Verfügung gestellt. Dieses Dokument beschreibt wie die Installation und wie das Bundle genutzt wird.

Um zu beginnen, stelle erst einmal fest, ob dein System richtig eingestellt ist. Obgleich die Graphiktreiberkonfiguration sehr tricky sein könnte, ist die tatsächliche Installation von LG3D selbst sehr einfach. Zum konfigurieren, installieren und laufen von LG3D folge den Schritten:

1. Systemvoraussetzungen

Bevor du dein Projekt Looking Glass starten kannst, solltest du schauen das du die minimalen Systemvorraussetzungen erfüllst um einen Problemlosen Start zu gewährleisten.

Komponente Voraussetzungen
Betriebssystem

Linux x86: Die meisten aktuellen Linux Distributionen sollten laufen.

Solaris x86: Unterstützt Solaris x86 S10_b74 (oder neuer).

CPU 1.4 GHz oder schneller empfohlen
RAM 512MB empfohlen
Graphikkarte

3D beschleunigte Graphikkarte, mit mindestens 32MB VRAM und Treiberunterstützung für OpenGL, Version 1.3 oder h&ounl;her (LG3D kann mit OpenGL 1.2 laufen, aber 1.3 wird unbedingt empfohlen). Anzeige 24bit Tiefe wird benötigt, damit die Integration mit X11 richtig arbeiten kann.

Linux x86: Für Nvidiakarten muss mindestens der Treiber in der Version 1.0-6629 verwendet werden. Lies dazu hier für mehr informationen. Es ist sehr wichtig diese Infos zu lesen wenn du auf diesen Treiber wechselst, da LG3D eine bestimmte Konfiguration des Treibers zum laufen benötigt.

Solaris x86: Zu der Zeit dieses Schreibens sind die einzige Art der Graphikkarten mit OpenGL Unterstützung auf Solaris x86 bestimmte Nvidia Karten (unten zu sehen) Du mußt mindestens die folgende Treiberversion verwenden : 1.0-8756 (oder neuer). Schaue hier für mehr informationen. Momentan werden nur die folgenden Nvidia Graphikarten durch den Solaris x86 Treiber unterstützt:

  • Quadro FX 4000
  • Quadro FX 3000
  • Quadro FX 1100
  • Quadro FX 500
  • NVS 280

If you are successful in getting LG3D to run on other cards, or want to report problems with specific cards, please let us know using the discussion forum

2. Downloaden & Installieren von LG3D

Das Projekt, Looking Glass, ist in einer Anzahl von Formaten vorhanden, einschließlich Debian .deb Pakete, binäre Bundle und Solaris Pakete. Die binären Bundle und die Debian Pakete enthalten alles, das erfordert wird, um lg3d (JRE, Java3D und LG3D) auf einem OS laufen zu lassen, das das entsprechende OpenGL installiert hat.

Auf Debian basierende Linux Distros (wie Ubuntu) wird empfohlen, daß du die Debian Pakete (.deb) benutzt, um LG3D zu installieren. Auf den anderen Linux Distros und Solaris x86 ist die bevorzugte Weise des Downloadens und des Installierens von LG3D, die Bundle zu benutzen. Das LG3D Bundle bringt alles mit, das du zum laufen lassen von LG3D benötigst, einschließlich das JRE und das Java 3D. Obgleich es recht groß ist, vereinfacht es deinen Installations Prozeß.

Andere LG3D Versionen können in der "Stable build section" gefunden werden: Die Binär Pakete Seite . Für Veröffentlichungsdetails, schaue bitte in die Release Notes.

2.1 Installieren von LG3D mit dem Megabundle

Der LG3D Mega-bundle Installer installiert LG3D und alle benötigten Komponenten inklusive JRE and Java 3D. Beachte das der Installer JRE und Java 3D in ein LG3D spezifisches Veriechnis (/usr/share) kopiert. Der Vorteil ist, das somit keine veränderungen an der Bestehenden Java Umgebung die du nutzt vorgenommen werden.

  1. Download das passende Megabundle von den folgenden Links :
    • LG3D Release 1.0 Linux x86 Mega Bundle (19 December 2006)

    • LG3D Release 1.0 Solaris x86 Mega Bundle (19 December 2006)
  2. Suche das herunter geladene Mega-bundle *.bin file, z.B. /tmp/lg3d-...-.bin

  3. Wechsle das aktuelle Verzeichnis in das, wohin du LG3D installieren möchtest.

    Linux Hinweis: Es ist wichtig das du LG3D in /usr/share auf deiner Maschine installierst. Du kannst es auch irgendwo anders hin installieren, nur dann wird deine gdm session kein LG3D starten können.


  4. Starte das Mega-bundle *.bin file, z.B.: sh /tmp/lg3d-...-.bin

    Linux Hinweis: Wenn du dich entschieden hast deine Installation in /usr/share zu machen, dann must du das Kommando als root user ausführen.

    Das Kommando erstellt ein Unterverzeichnis mit den Namen "lg3d" in dem aktuellen Verzeichnis und entpackt alles was du benötigst um LG3D laufen zu lassen (einschließlig JRE und Java 3D).


  5. Der Installer zeigt die Lizensvereinbarungen. Lese es und drücke dann die "Leertaste" um zu scrollen, danach beantworte die Frage mit einem "yes" oder "no".

  6. Wenn du "yes" drückst, fängt der Installer an alles in das "lg3d" Unterverzeichnis zu entpacken. Wenn es fertig ist, steht "Done.", sobald alle Dateien entpackt sind.

  7. Füge LG3D als eine gdm Session hinzu (nur Linux):

    8. Auf Linux Maschinen kannst du festlegen, damit LG3D direkt vom gdm Desktop gestartet werden kann. In diesen Modus übernimmt LG3D den gesamten Schirm und läuft als das alleinige Fenster-Verwaltungsprogramm.

    Um das einzustellen, wechsle zum user "root" und gebe folgende Kommandos ein:

    # cd /usr/share/lg3d/bin
    # ./postinstall

    Hinweis: Du kannst das nur so ausführen, wenn du LG3D nach /usr/share entpackt hattest.

  8. An dieser Stelle ist die Mega-bundle Installation bereits beendet.

2.2 Installieren der Debian .deb Pakete

Die Debian Pakete wurden auf einem Ubuntu Dapper System gebaut und getestet, sie sollten dennoch auf anderen Debian basierenden Systemen laufen.

Du kannst auf zwei Wegen die Debian Pakete herunterladen und installieren:
  1. Downloade die drei .deb Dateien von der Binär Pakete Seite und installiere dann mit dem folgenden Kommando:

    % sudo dpkg -i lg3d_jdk*.deb lg3d_java3d*.deb lg3d-core*.deb

    Hinweis: Die Installation der lg3d-core*.deb Pakete fügt automatisch LG3D als eine Session im gdm Desktop Session Wechsler hinzu. Es ist nicht nötig irgend ein Script von Hand auszuführen.

    2. ODER

  2. Installation durch Nutzung von apt oder dem Synaptic Package Manager (auf Ubuntu) indem folgendes gemacht wird (du must dich als root anmelden, mache das mit sudo bash als erstes) :
    • Da gibt es drei 3 LG3D repositories. Das stable repository besitzt die letzten stabilen Veröffentlichungen (0.8.1, 1.0 und so weiter). Das testing repository enthält die pre-release Versionen (alpha, beta und so weiter) für die letzten Versionen und das unstable repsoitory enthält die "nightly builds". Um diese Repositories zu benutzen, füge folgende Zeilen in /etc/apt/sources.list hinzu und kommentiere/unkommentiere die entsprechenden Zeilen. 
        	## LG3D repsoitories
  	deb http://javadesktop.org/lg3d/debian stable contrib
  	# deb http://javadesktop.org/lg3d/debian testing contrib
  	# deb http://javadesktop.org/lg3d/debian unstable contrib
    • Dann kannst Du 
                % apt-get update
  	  % apt-get install lg3d-core
      ausführen ODER, auf Ubuntu Systemen,
      • den Synaptic Package Manager im System/Administration Menü nutzen,
      • click auf den Relaod/Aktuallisieren Button und dann auf
      • Search/ Suchen für lg3d und markiere das lg3d-core Paket für die Installation.
      • click auf den Apply/ Übernehmen Button zum downloaden und installieren

    Das auswählen von lg3d-core für die Installation sollte automatisch lg3d-jdk und lg3d-java3d für die Installation mit auswählen. Du must die drei Lizenzen bestaätigen für lg3d, jdk und java3d um die Installation abzuschliessen.

3. LG3D starten

Es gibt die folgenden Möglichkeiten um LG3D laufen zu lassen:

  1. Developer mode (lg3d-dev) : Dieser Modus ist auch bekannt als "dev mode". In diesem Modus, läft LG3D in einem Fenster auf deinem Desktop. Dies ist der einfachste Weg um LG3D laufen zu lassen aber der Modus erlaubt dir nur Project Looking Glass 3D Anwendungen zu starten, so wie den CD viewer. Native X11 Anwendungen, zum Beispiel xterm starten ausserhalb des LG Fensters; sie erscheinen im X Fenster, von dem aus du LG3D gestartet hast.
    Um im dev mode zu starten:

    1. Prüfe ob OpenGL 1.3 oder neuer unterstützt wird:

      % glxinfo | grep "GLX version"
      GLX version: 1.3
      Wenn du einen Wert kleiner als 1.3 siehst, update deinen Graphiktreiber indem du der Beschreibung im       Abschnitt für die Graffikkartenvorraussetzungen. LG3D läft mit OpenGL 1.2, aber 1.3 wird dringend empfohlen.

    2. Wechsel das Verzeichnis in das LG3D Installationsverzeichnis.

    3. Starte den dev mode durch Eingabe von:

      % bin/lg3d-dev

    Wenn die LG3D Installation erfolgreich war, solltest du einen Startbildschirm und dann den LG3D Desktop in einem Fenster sehen. Wenn du auf das CD Icon in der Taskleiste klickst, öffnet sich der CD Betrachter, eine 3D Demo Anwendung. Wenn du mit der Maus über den Stapel der CDs gehst und auf eines der CD Bilder klickst, siehst du die Anwendung in Aktion. Für weiterführende Informationen, siehe bitte auch das LG3D erster Start Dokument (bitte beachte das "X Apps Specific Features" ist nicht Verfügbar im dev mode). Wenn du einige Dinge vermißt, konsultiere den Problembehebungs Abschnitt.


  2. Application Mode (lg3d-app) : Dieser Modus ist auch bekannt als "app mode." In diesem Modus läuft LG3D auch in einem Fenster auf deinem Desktop aber er erlaubt den nativen X11 Anwedungen im gleichen Fenster wie LG3D zu laufen, so wie die LG3D Anwendungen.
    Um LG3D in dem app mode zu starten:

    1. Prüfe ob OpenGL 1.3 oder neuer unterstützt wird:

      % glxinfo | grep "GLX version"
      GLX version: 1.3
      Wenn du einen Wert kleiner als 1.3 siehst, update deinen Graphiktreiber indem du der Beschreibung im       Abschnitt für die Graffikkartenvorraussetzungen. LG3D läft mit OpenGL 1.2, aber 1.3 wird dringend empfohlen.

    2. Wechsel in das Verzeichnis in dem du LG3D installiert hast.

    3. Starte den app mode durch Eingabe:

      % bin/lg3d-app

    Im Unterschied zum dev mode, solltest du nun alle X Anwendungen innerhalb des LG3D Fensters starten können. Teste xterm durch klicken des Terminal Icon in der Taskleiste. Du solltest ein xterm aufgehen sehen, innerhalb der 3D Umgebung. Wenn du einige Dinge vermißt, konsultiere den Problembehebungs Abschnitt.

    Wenn du möchtest, kannst du das LG3D Fenster als Vollbild starten durch Nutzung der "-f" Option:

    % bin/lg3d-app -f

    In diesem Modus läft lg3d-app unter dem GNOME Fenstermanager, so das du Zugriff zu der GNOME Taskbar und zu den Anwengungen hast die enthalten sind, aber das LG3D Fenster belegt den gesamten Bildschirm. Du hast keine Möglichkeit die Größe zu verändern.

    Wenn du das LG3D Fenster weiter configurieren möchtest, siehe auch das Resizing and Configuring LG3D Window Dokument .

    Linux Hinweis: wenn du LG3D in /usr/share/lg3d installiert hast und und dann postinstall hast laufen lassen während der Installation , dann hast du eine andere Möglichkeit den app mode zu starten. In dieser Einstellung wurde LG3D als eine andere Desktop Session in dem gdm Desktop Session Wechsler eigetragen. Wenn das System neu startet und der Session Wechsler zur Verfügung steht, klicke auf "Session" und dann klick in dem Menü auf "Looking Glass". In diesem Modus starte Gnome nicht mit, und LG3D ist der alleinige Fenstermanager.

Entdecke den Looking Glass Desktop und beginne mit den Trämen bon neuen Wegen den Desktop in 3D zu nutzen! Wenn du bereit bist zum Entwickeln, schaue über die Project Looking Glass Developer's Guide Seite für Informationen über das entwickeln mit dem Project Looking Glass.

APPENDIX A: Color Depth Configuration

Java Desktop System 1 and 2, and SUSE8.1:

  1. From within a desktop session, run the sax2 command:

    % /usr/X11R6/bin/sax2
    SaX: root Password:     enter root password

  2. In the SaX2 application, click on Color and Resolution then click the [Properties...] button for the currently configured desktop.
  3. In the Color selection... panel, click on the the menu and choose 16.7 Mio. [24 Bit].
  4. On the Resolution(s) for 16.7 Mio. [24 Bit] colors tab, select the preferred resolution for your desktop and press [Ok].
  5. Click the [Finish] button, then [Finalize>>].
  6. The SaX2 final steps... dialog will open. Click the [Test...] button to test the new resolution.
  7. A blue test screen should appear, click the [Save] button if the display appears to be stable, and then [Ok] when you return to the SaX2 window, followed by [Yes] to exit from SaX2.
  8. If the test screen did not appear correctly, the test will exit after 30 seconds and your previous desktop session will be restored. You should then try alternate resolutions at the 24 bit color depth to find a resolution that works for your system.
  9. When you are satisfied with the settings, log out and re-login to your desktop session and verify that the color depth is now 24 bit or higher using the xdpyinfo command above.

RedHat Fedora 2:

  1. First become root:

    % su root
    Password:     enter root password
    #

  2. Edit the file: /etc/X11/xorg.conf
  3. Locate the section of the file labeled: Section "Screen"
  4. Within this section, locate the entry: Identifier "Screen0"
    Change the DefaultDepth and DefaultFbBpp properties to:

    DefaultDepth 24
    DefaultFbBpp 32

  5. Save the file and then logout and re-login to your desktop session and verify that the color depth is now 24 bit or higher using the xdpyinfo command above.

RedHat 8, RedHat 9 and other Linux distributions:

Many Linux platforms use the XF86Config file to configure the display properties. To change the color depth on these systems:

  1. First become root:

    % su root
    Password:     enter root password
    #

  2. Edit the file: /etc/X11/XF86Config
  3. Locate the section of the file labeled: Section "Screen" and change the DefaultDepth property to 24:

    DefaultDepth     24

  4. Within this section, locate the sub-section labeled "Display":

    SubSection "Display"

    and change the Depth property to:

    Depth     24

  5. Save the file and then logout and re-login to your desktop session and verify that the color depth is now 24 bit or higher using the xdpyinfo command above.

Solaris x86 :

Solaris x86 uses the xorg.conf file to configure the display properties for the Xorg server which is used by Project Looking Glass. To change the color depth on Solaris x86:

  1. First become root:

    % su root
    Password:     enter root password
    #

  2. Edit the file: /etc/X11/xorg.conf
  3. Locate the section of the file labeled: Section "Screen" and change the DefaultDepth property to 24:

    DefaultDepth     24

  4. Save the file and then logout and re-login to your desktop session and verify that the color depth is now 24 bit or higher using the xdpyinfo command above.


APPENDIX B: Bringing Down X Session

The instructions for shutting down your X session vary by platform, so follow the instructions that apply to your platform. If your platform isn't listed, chances are the process will be very similar. So, use these instructions as a guide and let us know what's different!

Sun Java Desktop System/SuSE 8.1:

JDS/SuSE 8.1 uses the X Display Manager (xdm) to manage user sessions. To run Project Looking Glass you must first shut down xdm. Since this process is respawned automatically you must disable this service using the chkconfig command:

  1. First become root:

    % su root
    Password:     enter root password
    #

  2. Disable the xdm service:

    # /sbin/chkconfig -d xdm
    xdm  0:off   1:off   2:off   3:off   4:off   5:off   6:off

  3. Now close any desktop applications and save any open files. Then shutdown the display manager:

    # pkill gdm-binary

  4. Now skip to the Starting the Project Looking Glass desktop session section.

Note: To re-activate the X Display manager, enter the following command as root, and then reboot your system:

# /sbin/chkconfig -a xdm

RedHat 9:

RedHat uses the GNOME Display Manager (gdm) to manage user sessions. It's not necessary to disable gdm, you can simply switch to a lower run level where gdm is not active:

  1. First become root:

    % su root
    Password:     enter root password
    #

  2. Close any desktop applications and save any open files. Then switch to run level 4:

    # /sbin/init 4

    Your X session should terminate and your should return to the console prompt.

  3. Now skip to the Starting the Project Looking Glass desktop session section.

Note: To restart the GNOME Display Manager, enter the following command as root from the console:

# /sbin/init 5

Fedora Core 3 64-bit:

To run Project Looking Glass full screen on the 64-bit version of Fedora Core 3 requires a few unique changes. Note that you will need access to a 32-bit Linux system or the 32-bit X11 modules for Linux to complete this procedure.

  1. You will need to copy the contents of the /usr/X11R6/lib/modules directory from a 32-bit Linux system to the same directory on your 64-bit system. Before performing this action, make a backup of the /usr/X11R6/lib/modules directory on your 64-bit system:

    # cd /usr/X11R6/lib
    # cp -r modules modules.orig

    Next, use the scp command to securely copy the files from the 32-bit system to your 64-bit system:

    # scp -r root@32bithost:/usr/X11R6/lib/modules /usr/X11R6/lib/modules

    where 32bithost is the host name of the 32-bit Linux system.

  2. Make sure that your xorg.conf file is set for depth of 24 as shown in bold below:

    Section "Screen"
    Identifier "Screen0"
    Device "Videocard0"
    Monitor "Monitor0"
    DefaultDepth 24
    SubSection "Display"
    Viewport 0 0
    Depth 24
    Modes "1680x1050" "1280x800" "1024x768" "800x600" "640x480"
    EndSubSection
    EndSection
  3. Proceed to the next section to start the full screen session.

    [Instructions in this section courtesy of qgonjon and marve. See this support thread.]

Solaris x86:

In order to run Project Looking Glass in session mode on Solaris x86, you must first shut down CDE and disable its subsequent boot-time start up. To do this, become root and type:

# /usr/dt/bin/dtconfig -d

then reboot your computer.



APPENDIX C: Troubleshooting

Screen updates are very slow

This usually means that graphics acceleration is not correctly configured, or the graphics card is not capable enough to run LG3D. Here are some things to check first:

  • Run the glxinfo command from within an X11 session and check that the direct rendering property is set to Yes. Also look for references to OpenGL to ensure that your graphics accelerator device driver supports OpenGL.
    - For NVIDIA cards, run xdpyinfo and verify that you see GLX or NVIDIA-GLX.

    If you don't see these properties then you probably don't have the appropriate device driver installed or 3D enabled. Refer to your vendor's web site for information on installing and configuring a device driver with 3D/OpenGL support.

  • Check the X server log file at /var/log/Xorg.0.log for error messages and to check which drivers are being loaded.
Even if it turned out that the graphics card is less capable, you may be able to run LG3D with 16bit color. Please refere to the Color Depth Configuration instructions. Note that the session mode in 16bit color is not fully supported, and you may encounter rendering and other issues.

Fatal Server error: Cannot establish any listening sockets

The following error message is displayed when you start Project Looking Glass using lg3d-session:

_XSERVTransSocketUNIXCreateListener: ...SocketCreateListener() failed
_XSERVTransMakeAllCOTSServerListeners: server already running

Fatal server error:
Cannot establish any listening sockets - Make sure an X server
isn't already running

First, make sure that you do not have an X11 session (or an lg3d-app window) running on the display on which you're starting lg3d-session. Then check to see if you have /tmp/.X11-unix/X0 (or X1 for lg3d-app) left behind from a previous session. To fix this problem, delete this file and restart lg3d-session (or lg3d-app).

SEVERE: Unable to listen on Socket 44816. Make sure to terminate all the other LG3D processes.

You can get this even if you are sure there is no other LG3D process running. It typically happens when the previous LG3D session was terminated in an abnormal manner. Do the following and kill all the Java processes. Be careful to close the other running Java apps like NetBeans and Eclipse before performing this:

% pkill -9 java

An unexpected exception has been detected in native code outside the VM

Project Looking Glass exits with the following exception, even though you've verified that you have 3D acceleration enabled:

An unexpected exception has been detected in native code outside the VM.
Unexpected Signal : 11 occurred at PC=0x0
Function=[Unknown.]
Library=(N/A)

Check to see if the following entries appear in the exception message:

<addr-range> r-xp 00000000 03:05 61763 /usr/lib/GL/libGL.so.1.3.mesasoft
<addr-range> rw-p 001b4000 03:05 61763 /usr/lib/GL/libGL.so.1.3.mesasoft

If so, you have the MESA software OpenGL libraries installed. Use the package tool for your distribution (e.g., rpm) to remove these libraries, and then go into /usr/lib as root and check that the following symbolic links exist and point to the correct OpenGL libraries. You should have links that look something like this (where x and y are numbers):

libGLcore.so.1 -> libGLcore.so.1.x
libGL.so -> libGL.so.x
libGLU.so -> libGLU.so.x.y.z

If these links are not correct, become root and create them. For example, if you have a libGLU.so.1 file but no libGLU.so link, then create the link:

# ln -s /usr/lib/libGLU.so.1 /usr/lib/libGLU.so

Repeat this for each of the links that are missing.

Fatal server error: could not open default font 'fixed'

In some Linux distro, the /usr/X11R6/lib/X11/fonts direcotry doesn't exist, or it doesn't include font files that LG3D's X server expects. Do the followings and see if it fixes the issue:

% sudo mv /usr/X11R6/lib/X11/fonts /usr/X11R6/lib/X11/fonts.orig
% sudo ln -s /usr/share/X11/fonts /usr/X11R6/lib/X11/fonts

libGL error: failed to open DRM: operation not permitted

(This error may be encountered when running lg3d-dev).

If Project Looking Glass runs very slowly, even though 3D acceleration is enabled, check to see if the following message appears in the console:

libGL error: failed to open DRM: Operation not permitted
libGL error: reverting to (slow) indirect rendering

This means that your user account doesn't have permission to use the DRI (Direct Rendering Infrastructure). In some cases root is the only user with access to Direct Rendering. This problem seems to only affect RedHat users, and not JDS/SuSE users. If you have this problem, you should change the DRI permissions in your xorg.conf or XF86Config file to:

Section "DRI"
Group "video"
Mode 0666
EndSection

You will also need to change the permissions of the <dri> profile in /etc/security/console.perms from 0600 to 0666:

<xconsole> 0600 <dri> 0666 root

and if you're using an NVIDIA card, change the permissions on /dev/nvidia* to 0666:

# chmod a+rw /dev/nvidia*

Failed to load the NVIDIA driver

There are several possible reasons for the driver failing to initialize correctly:

  • The NVIDIA driver is compiled for linux.*bigmem. If your Linux system has two images, bigmem and linux*up, use the former one. Otherwise the driver will fail to load.

  • The NVIDIA driver needs to find the IRQ that is assigned to the video card. This should be set by the BIOS before the OS boots, but it can be switched off in some BIOS. Try switching it back on again to see if it helps.

  • Refer to the NVIDIA driver installation instructions and release notes for further details.

no core pointer

This happens on RedHat Fedora Core 2, and may happen on other configurations as well. One known workaround (from sgiunchi) is to do the following (as root user):

# ln -s /etc/X11/xorg.conf /etc/X11/XF86Config-4

The correct long term fix is to upgrade the X11 release which is included in lg3d-core. It is currently an Xorg 6.7 pre-release and it needs to be upgraded to the latest Xorg stable release.

Exception in thread "main" java.lang.UnsatisfiedLinkError: /usr/java/jdk1.5.0/jre/lib/i386/libj3dcore-ogl.so:
/usr/java/jdk1.5.0/jre/lib/i386/libj3dcore-ogl.so: symbol glMultiDrawArraysEXT, version LIBGL not defined in file libGL.so.1 with link time reference

This error occurs with ATI device drivers that are missing the glMultiDrawArraysEXT extension. The solution to this problem is to replace the Radeon device drivers with drivers from the Direct Rendering Open Source (DRI) project. For instructions on installing direct rendering drivers refer to DRI driver installation for ATI Radeon.

Note that upgrading libGL.so to the latest Mesa libraries does not solve this problem.

pkill: 27863 - Operation not permitted
java.rmi.server.ExportException: Port already in use: 1099; nested exception is:
      java.net.BindException: Address already in use...

This message indicates that an rmiregistry process is already running and lg3d-dev was unable to start a new instance. Typically this occurs when the rmiregistry process was started by root and lg3d-dev is run as a non-root user. The solution to this problem is to stop the root instance of rmiregistry. Become root and stop the process:

% su root
Password: enter root password
# pkill rmiregistry
# exit

Now re-run the lg3d-dev command as a non-root user to start Project Looking Glass.
